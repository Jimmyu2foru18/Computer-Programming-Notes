DSA II: Page 8 - Graph Traversal (BFS & DFS)

Exploring Complex Relationships

A graph is just a collection of nodes and edges. To actually use that data, you need to be able to visit those nodes in a systematic way. This is called Graph Traversal. There are two primary approaches: Breadth-First Search (BFS) and Depth-First Search (DFS). Choosing between them often depends on what you're trying to achieve—finding the shortest path (BFS) or exploring every branch of a structure (DFS).

1. BFS: Breadth-First Search

BFS explores the graph level-by-level. Starting at the root, it visits all immediate neighbors, then all neighbors of neighbors, and so on. BFS is implemented using a Queue (FIFO), which keeps track of the next nodes to visit.

Use Case: Finding the shortest path in an unweighted graph (because it expands in concentric circles).


def bfs(graph, start):
    visited = {start}
    queue = [start]
    while queue:
        node = queue.pop(0)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

2. DFS: Depth-First Search

DFS explores the graph by going as deep as possible down one branch before backtracking. It is implemented using a Stack (or recursion), which naturally tracks the path backwards.

Use Case: Cycle detection, solving mazes, topological sorting.

Real-World Relationship: Searching for a Lost Key

Think of BFS like searching for a lost key in a Multi-Room House by looking in every room on the ground floor first, then every room on the second floor. You are guaranteed to find the key in the fewest *number of rooms* away from your starting point. Think of DFS like searching by following one specific path—going all the way to the attic, then checking the attic closets, then coming down to the bedroom, then the bedroom closet. If the key is in the attic, you find it instantly, but if it was in the kitchen (the very last place you look), you waste a lot of time exploring other rooms first.
