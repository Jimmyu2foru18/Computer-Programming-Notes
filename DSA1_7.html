DSA I: Page 7 - Binary Search

The Power of Divide and Conquer

Binary Search is a vastly more efficient search algorithm than linear search, but it comes with one strict requirement: the dataset must be sorted. Instead of checking every element one by one, binary search uses a "divide and conquer" approach, repeatedly splitting the search range in half. This dramatically reduces the number of comparisons needed, turning an O(n) task into an O(log n) task.

# Binary Search: Requirement = Sorted Array
sorted_arr = [1, 3, 5, 7, 9, 11]
target = 7 # Efficiently find index using mid-point splits

1. The Algorithm

The core logic is to ignore half the remaining search space with each comparison:

  1. Find the middle element of the current range.
  2. If the middle element is the target, you're done!
  3. If the target is less than the middle element, repeat the search in the left half.
  4. If the target is greater than the middle element, repeat the search in the right half.
  5. If the search range is empty and you haven't found the target, it's not in the array.
# Standard Iterative Binary Search
def binary_search(arr, target):
    low, high = 0, len(arr) - 1
    while low <= high:
        mid = (low + high) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1

2. Why It Scales So Well (O(log n))

Because you cut the search space in half with every single step, the number of steps required grows very slowly. For an array of 1,000,000 items, Linear Search might take 1,000,000 checks, but Binary Search will take at most 20 (because 2^20 is approx. 1,000,000). This exponential reduction in steps is what makes it so fast.

# Demonstrating logarithmic scale: O(log n)
def log_steps(n):
    steps = 0
    while n > 1:
        n //= 2
        steps += 1
    return steps # For n=1,000,000, steps is ~20

Real-World Relationship: Finding a Name in a Phonebook

Think of Binary Search like finding a name in a Physical Phonebook. Because the names are already sorted alphabetically, you don't start at the very first page and read every single name (Linear Search). Instead, you open the book to the middle. If the name you are looking for is later in the alphabet, you ignore the entire first half and open the remaining half to its middle. You repeat this, cutting the book in half each time, until you find the name. The sorted order is what makes this "divide and conquer" approach possible.
