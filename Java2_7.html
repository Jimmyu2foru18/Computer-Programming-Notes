Java II: Page 7 - Robustness: Exception Handling

Expecting the Unexpected

In the real world, things go wrong. A user might enter text when a number was expected, a file you're trying to read might have been deleted, or a network connection might drop. If your program doesn't know how to handle these situations, it will "crash" (terminate abruptly with an error message). Exception Handling is Java's mechanism for catching these errors at runtime and deciding how to recover, ensuring your application remains stable and user-friendly.

1. The Exception Hierarchy

In Java, all exceptions are objects. They all descend from the Throwable class. There are two main categories of exceptions you must understand:

  • Checked Exceptions: These are checked at compile-time. Java forces you to handle them (e.g., IOException). Think of these as "recoverable" errors.
  • Unchecked (Runtime) Exceptions: These occur during execution and are usually the result of programming bugs (e.g., NullPointerException, ArrayIndexOutOfBoundsException).

2. The try-catch-finally Block

To handle an exception, you "try" to run a block of code and "catch" any errors that occur. The finally block is optional and runs no matter what, making it perfect for cleanup tasks like closing a database connection.


try {
    int result = 10 / 0; // Throws ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Error: Cannot divide by zero!");
} finally {
    System.out.println("Cleanup completed.");
}

Try-with-resources (Java 7+)

Modern Java allows you to declare "closable" resources directly in the try header. Java will automatically close them for you when the block finishes.


try (Scanner scanner = new Scanner(new File("test.txt"))) {
    // Process file
} catch (FileNotFoundException e) {
    e.printStackTrace();
}

3. Throwing & Custom Exceptions

You can use the throw keyword to trigger an exception manually. If a method might throw a checked exception, it must declare it using the throws keyword in its signature. You can also create your own specialized exceptions by extending the Exception class.


public void setAge(int age) throws InvalidAgeException {
    if (age < 0) {
        throw new InvalidAgeException("Age cannot be negative!");
    }
}

Real-World Relationship: The Safety Net and the Hospital Backup

Think of Exception Handling like a Trapeze Safety Net. The circus performers (your code) are doing dangerous, high-stakes work. Most of the time they land perfectly. But if they slip (an error), the net (the try-catch) catches them so the show can go on, rather than letting them hit the ground (Crashing the program). Think of Checked Exceptions like a Pre-Flight Checklist for a pilot: the system forces you to check for problems (fuel, engines, weather) before you are even allowed to take off. Finally, think of the finally block like a Hospital's Sterilization Protocol: no matter if a surgery was a success or had complications, the room must be cleaned and tools sterilized for the next patient. This cleanup is non-negotiable for system health.
