Software Engineering II: Page 5 - Modern Operations: DevOps & CI/CD

The Velocity of Software

In the past, developers would write code for months and then "throw it over the wall" to the Operations team, who were responsible for deploying and maintaining it. This created friction, as developers wanted new features while Operations wanted stability. DevOps is a cultural and professional movement that breaks down these silos. It combines Development (Dev) and IT Operations (Ops) to shorten the systems development life cycle and provide continuous delivery with high software quality. CI/CD is the technical engine that makes DevOps possible.

1. Continuous Integration (CI)

Continuous Integration is the practice of automating the integration of code changes from multiple contributors into a single software project. Whenever a developer "pushes" code to a shared repository (like GitHub), an automated system immediately builds the project and runs a suite of Unit Tests. If the tests fail, the developer is notified instantly, preventing "broken" code from ever reaching the main branch.

2. Continuous Delivery & Deployment (CD)

CD picks up where CI leaves off. It automates the delivery of integrated code to environments like testing or staging. Continuous Deployment goes one step further: every change that passes all stages of your production pipeline is released to your customers automatically, without human intervention. This allows companies like Netflix or Amazon to deploy code thousands of times per day.

The Pipeline Stages:

  1. Source: Code is committed to the repository.
  2. Build: The application is compiled and dependencies are gathered.
  3. Test: Automated tests (Unit, Integration, Security) are run.
  4. Deploy: The code is moved to a server (Staging or Production).

3. Infrastructure as Code (IaC)

In a DevOps world, even the servers themselves are treated like software. Instead of manually clicking buttons in a cloud console, engineers write code (using tools like Terraform or Ansible) that describes the infrastructure. This makes server setup repeatable, version-controlled, and immune to human error.

Real-World Relationship: The Assembly Line and the Quality Inspector

Think of CI/CD like a Modern Car Assembly Line. In the old days, you'd build a whole car and then check if it worked. In a modern factory, every single part is scanned by a laser (CI/Automated Testing) the second it is installed. If a bolt is loose, the line stops immediately. Once the car is finished, it is automatically driven onto a transport truck (CD/Deployment). Think of Infrastructure as Code like a 3D Printer. Instead of manually carving a piece of wood, you write a digital file (the Code) that tells the printer exactly what to make. If you want 100 identical pieces, you just hit "Print" 100 times. You don't have to worry about one being slightly different than the other because the "Instructions" are perfectly preserved in the code.
