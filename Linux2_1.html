Linux II: Page 1 - Networking Commands

Connectivity & Remote Management

Linux is the operating system of the internet, powering the vast majority of web servers and cloud infrastructure. Therefore, understanding how Linux interacts with networks is a critical skill for any system administrator. Whether you are configuring a local interface, debugging a connectivity issue, or remotely managing a server on the other side of the planet, the command line provides the precise tools needed to get the job done efficiently and securely.

1. The Networking Toolkit

Networking commands allow you to inspect, configure, and troubleshoot network traffic:

  • Interface Management: ip addr (the modern replacement for ifconfig) lets you view and manage network interfaces. ip route manages the routing table.
  • Connection Analysis: ss (Socket Statistics) is the modern, faster replacement for netstat. It allows you to see all active network connections, listening ports, and sockets.
  • Remote Access: ssh (Secure Shell) is your primary tool for logging into remote servers securely. scp (Secure Copy) uses SSH to securely copy files between machines.
  • Diagnostics: traceroute shows the path packets take to a destination; nslookup and dig are essential for querying DNS servers; curl and wget are powerful tools for transferring data from or to servers (testing APIs or downloading files).

# View network interfaces
ip addr show
# See all listening TCP ports
ss -tuln
# Remotely log into a server
ssh user@remote-server.com

2. Why Command Line Networking?

Network configuration in production environments is almost exclusively managed via the CLI. It allows you to script configuration changes, verify connectivity with precise diagnostic tools, and manage remote systems without the overhead or security risks of a graphical interface. Mastery here is the bridge between managing a single computer and managing a data center.

Real-World Relationship: The Network Technician

Think of Networking Commands like the Diagnostic Gear of a Network Technician. When you have a problem with your home Wi-Fi, you might just restart the router. But when a massive server cluster goes down, you need to plug in your diagnostic tools (CLI commands) to "listen" to the traffic and find out exactly where the connection is failing. ssh is like having a direct telephone line to that remote server, scp is the secure delivery driver, and traceroute is the detailed map showing you which intersection the traffic jam is happening at.
