Linux I: Page 9 - Redirection & Piping

The Power of Chaining Commands

In Linux, the true power of the command line lies in the ability to chain simple, single-purpose tools together to perform complex tasks. This is made possible by I/O Redirection and Piping. By default, every command takes input from your keyboard (Standard Input, or stdin) and sends its output to your screen (Standard Output, or stdout). Redirection and piping allow you to take that output and feed it into a file or another command instead, creating powerful automated workflows.

1. Redirection Operators

  • >: Redirects output to a file (overwrites existing content).
  • >>: Redirects output to a file (appends to existing content).
  • <: Takes input from a file instead of the keyboard.

# Save file list to a document
ls -l > file_list.txt

# Append current date to a log file
date >> activity.log

2. Piping (|)

The Pipe (|) operator takes the stdout of the command on the left and sends it as the stdin to the command on the right. This is how you build complex workflows.


# List files and immediately filter for those containing "log"
ls -l | grep "log"

Real-World Relationship: The Assembly Line

Think of Redirection like sending a finished product (the command output) into a shipping container (the file) for later use. Think of Piping like a Continuous Assembly Line. The output of one machine (Command1) isn't finished—it's just the raw material needed for the next machine (Command2) on the line. You can have a whole factory floor where dozens of machines are chained together, with the raw material (Standard Input) going in one end, undergoing dozens of transformations, and finally emerging as a finished product at the end of the line.
