Web Technologies II: Page 2 - PHP: Web & Databases

Dynamic Web Interaction

PHP is at its best when it interacts with the user and stores data. Web applications are "stateful," meaning they remember who you are (Sessions) and process the information you submit (Forms). By connecting PHP to a database, you can transform a static site into a dynamic one—like a blog, an e-commerce store, or a user dashboard.

1. Handling Forms: The Bridge

Forms are how users talk to your server. When a user clicks "Submit," the browser sends the form data to a PHP script using either GET (visible in the URL) or POST (hidden in the request body). In PHP, you access this data via Superglobal arrays.


// process.php
$name = $_POST['name'];
echo "Hello, " . htmlspecialchars($name);

Sessions: Remembering Users

HTTP is "stateless," meaning the server forgets you the moment a page loads. Sessions create a persistent connection by storing a unique Session ID in the user's browser (a cookie). PHP uses this ID to map requests to the correct data on the server.

2. Connecting to Databases (MySQL)

PHP communicates with databases (like MySQL) to save and retrieve data. You should always use PDO (PHP Data Objects) because it supports prepared statements, which are vital for preventing SQL Injection attacks.


// Connecting with PDO
$db = new PDO('mysql:host=localhost;dbname=test', 'user', 'pass');
$stmt = $db->prepare("SELECT * FROM users WHERE email = ?");
$stmt->execute([$email]);
$user = $stmt->fetch();

Real-World Relationship: The ATM

Think of Forms and PHP like an ATM Machine. You insert your card and enter your PIN (the Form Submission). The ATM (the PHP backend) takes your request, verifies your identity against the central banking server (the Database), and then returns your cash or tells you "Invalid PIN" (the Response). Think of a Session like the Active Session at the ATM. Once you log in, the ATM "remembers" it's you for that specific transaction until you hit "Cancel" or your time runs out. The bank doesn't have to re-verify who you are for every single button you press; it maintains that session until you explicitly end it.
