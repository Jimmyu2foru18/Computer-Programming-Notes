System Design III: Page 3 - Deployment: CI/CD

Automated Delivery

In a professional environment, you never manually copy files to a server. That is a recipe for error. Continuous Integration (CI) and Continuous Deployment (CD) (together known as CI/CD) automate the entire path from code change to live site. Every time you push to Git, a pipeline kicks off: running tests, building assets, and deploying your application automatically. If a test fails, the pipeline halts, protecting your production environment from breaking changes.

1. The Pipeline Components

  • CI (Continuous Integration): Automatically building and testing code on every push to detect bugs early.
  • CD (Continuous Deployment): Automatically shipping code to production servers if the CI tests pass.

# GitHub Actions Example
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm install && npm test
