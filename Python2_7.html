Python II: Page 7 - Object-Oriented Programming (OOP): Polymorphism & ABCs

The Art of Many Forms

In the previous pages, we learned how objects are built and how they inherit traits. Now, we come to one of the most powerful concepts in Object-Oriented Programming: Polymorphism. The word comes from Greek, meaning "many forms." In programming, it refers to the ability of different types of objects to be handled through a single, uniform interface. This allows you to write code that is flexible, scalable, and easy to extend without changing existing logic.

Duck Typing: Python's Secret Sauce

Most "strongly typed" languages (like Java or C++) require an object to explicitly inherit from a specific class to be considered compatible. Python, however, uses a concept called Duck Typing. The philosophy is simple: "If it walks like a duck and it quacks like a duck, then it is a duck."

In other words, Python doesn't care about the object's type; it only cares about what the object can do. If an object has a speak() method, you can pass it to a function that needs to make entities speak, regardless of whether that object is a Dog, a Cat, or a Robot.


class Dog:
    def speak(self): return "Woof!"
class Cat:
    def speak(self): return "Meow!"
class Robot:
    def speak(self): return "Beep Boop!"

def animal_concert(animals):
    for animal in animals:
        # Polymorphic call: the correct speak() is chosen at runtime
        print(f"{animal.__class__.__name__}: {animal.speak()}")

animal_concert([Dog(), Cat(), Robot()])

Abstract Base Classes (ABCs)

While Duck Typing is flexible, sometimes you want to enforce that certain classes follow a specific "contract." For example, you might want to ensure that every Shape class must have an area() method. For this, Python provides the abc module to create Abstract Base Classes. You cannot create an object from an abstract class; it serves only as a template for other classes to follow.


from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        """This must be implemented by all children."""
        pass

class Square(Shape):
    def __init__(self, side):
        self.side = side
    def area(self):
        return self.side * self.side

Real-World Relationship: The Universal Remote and the Electrical Outlet

Think of Polymorphism like a Universal TV Remote. The remote has a "Volume Up" button. It doesn't need to know if your TV is a Sony, a Samsung, or an LG—it just sends the "Volume Up" command (the Method Call), and each TV responds in its own specific way (the Implementation). Think of Abstract Base Classes like the Standards for an Electrical Outlet. The government (the ABC) doesn't build the appliances, but it sets a rule: "If you want to be a toaster or a vacuum, you must have this specific three-prong plug." By following this "contract," any appliance can get power from any outlet, ensuring the whole system works together seamlessly.
