Linux II: Page 5 - User & Group Management

Managing Multi-User Systems

Linux is designed to be a multi-user system from the ground up. This means the OS assumes that multiple people (or services) will be accessing the system simultaneously, each requiring different levels of access. User and Group Management is the framework that enforces this security. By organizing users into groups, you can efficiently manage permissions for dozens of users at once, rather than manually configuring access for every single individual account.

1. Core User Commands

  • useradd: Creates a new user account. Use the -m flag to create a home directory.
  • usermod: Modifies an existing user account (e.g., changing their shell, adding them to groups, renaming them).
  • userdel: Deletes a user. Use -r to remove their home directory as well.
  • passwd: Sets or updates a user's password.

# Create a user with a home directory
sudo useradd -m johndoe

# Add user to the sudo (admin) group
sudo usermod -aG sudo johndoe

2. Managing Groups

Groups are containers for users. By assigning permissions to a group, every user in that group automatically inherits those permissions.

  • groupadd: Creates a new group.
  • gpasswd: Administers group members (add/remove users).

Real-World Relationship: The Company Org Chart

Think of User Management like Onboarding Employees at a company. Each person needs a desk (home directory) and a specific set of tools to do their job. Group Management is like Assigning Employees to Departments (HR, Engineering, Marketing). Instead of giving every single employee individual keys for every single office door, you give the Engineering department group key access to the lab, and the HR department group key access to the records room. It's scalable, organized, and makes security audits much faster.
