Java I: Page 4 - Control Flow: Branching with If & Switch

Decision Making in Java

In the previous pages, we built a foundation of data and logic. Now, we must give our programs the ability to choose a path. Control flow is the order in which individual statements, instructions, or function calls are executed or evaluated. In Java, branching is achieved primarily through two structures: the flexible if-else block and the highly structured switch statement.

1. The if-else Statement

The if statement is the most basic control flow tool. It evaluates a boolean expression. if the result is true, the code block is executed. You can chain multiple conditions using else if and provide a final "catch-all" with else.


int score = 85;

if (score >= 90) {
    System.out.println("Grade: A");
} else if (score >= 80) {
    System.out.println("Grade: B");
} else {
    System.out.println("Grade: C or below");
}

Java also supports the Ternary Operator for simple one-line decisions: String status = (score >= 60) ? "Pass" : "Fail";

2. The switch Statement

When you need to compare a single variable against many possible discrete values, a switch statement is often cleaner and more performant than a long chain of if-else blocks. Traditionally, switch was used for simple equality checks on integers, chars, or Strings.

The Traditional Switch


int day = 3;
switch (day) {
    case 1: System.out.println("Monday"); break;
    case 2: System.out.println("Tuesday"); break;
    case 3: System.out.println("Wednesday"); break;
    default: System.out.println("Invalid day");
}
Warning: In traditional switches, forgetting the break keyword causes "Fall-through," where the program continues executing the next case regardless of whether it matches. This is a common source of bugs.

3. The Enhanced Switch (Java 14+)

Modern Java has introduced a much more powerful and safer version of the switch. It uses the "Arrow" (->) syntax, which automatically prevents fall-through and allows the switch to return a value directly.


String dayType = switch (day) {
    case 1, 2, 3, 4, 5 -> "Weekday";
    case 6, 7          -> "Weekend";
    default            -> "Unknown";
};

Real-World Relationship: The Railroad Switch and the Menu

Think of If-Else like a Railroad Switch. A sensor checks the weight or destination of the train (the Condition). If the train is heading to "City A," the track moves to the left. If it's heading to "City B," it moves to the right. Each decision leads to a completely different destination. Think of Switch like a Vending Machine Menu. You press button 1 for water, button 2 for soda, or button 3 for juice. You don't need to check "Is it 1? No. Is it 2? No. Is it 3?"—the machine just jumps straight to the item you requested. The Enhanced Switch is like a smart vending machine that doesn't just give you the item, but also updates your account balance and prints a receipt (returns a value) in one smooth, error-proof motion.
