C# I: Page 6 - Collection Types: Arrays & Basic Collections

Storing Groups of Data

Until now, we've used single variables to hold one piece of information. But what if you need to store 100 student scores, or a list of items in a shopping cart? In C#, we use Collections to manage groups of related objects. The most fundamental collection is the Array. While simple and fast, arrays have a fixed size. As you advance, you will also use dynamic collections like Lists, which can grow and shrink as needed. This page focuses on the core mechanics of storing and accessing sequences of data efficiently.

1. The C# Array: Fixed-Size homogeneous Storage

An array is a data structure that stores a fixed-size sequence of elements of the same type. Once an array is created, you cannot change its size. This makes it extremely memory-efficient but less flexible than other collections.

Declaration and Initialization


// Declare and set size later
int[] numbers = new int[5]; 
numbers[0] = 10; // Zero-based indexing

// Direct initialization (Array Literal)
int[] scores = { 95, 87, 92, 78, 88 };

Bounds Checking: C# is a safe language. If you try to access numbers[10] when the size is 5, the program will throw an IndexOutOfRangeException rather than corrupting memory.

2. Multidimensional and Jagged Arrays

C# supports 2D grids (Multidimensional) and "Arrays of Arrays" (Jagged).

  • Multidimensional: A perfect rectangle of data (e.g., int[,] matrix = new int[3,3];).
  • Jagged: A collection where each "row" can have a different length. This is more flexible and often faster for large datasets.

int[][] jagged = new int[3][];
jagged[0] = new int[4]; // Row 0 has 4 items
jagged[1] = new int[2]; // Row 1 has only 2 items

3. Introduction to Lists (Dynamic Arrays)

Because fixed-size arrays are often too restrictive, C# developers frequently use List<T>. Lists are "Dynamic Arrays"—they manage an internal array for you and automatically resize it when you add more items than it can currently hold.


using System.Collections.Generic;

List<string> names = new List<string>();
names.Add("Alice");
names.Add("Bob");
Console.WriteLine(names.Count); // 2

Real-World Relationship: The Egg Carton and the Accordion Folder

Think of an Array like an Egg Carton. When you buy a 12-egg carton, it has 12 physical slots (the Length). You can't suddenly add a 13th egg without getting a whole new carton. Every slot is designed for one egg (homogeneous data). Think of a List like an Accordion Folder. As you add more papers (data), the folder stretches and expands. You don't have to worry about the size when you start; the folder "manages" the physical space for you. This abstraction allows engineers to focus on the data logic rather than the low-level memory management of the computer.
