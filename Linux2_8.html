Linux II: Page 8 - Backups & Recovery

The Golden Rule of Admin: Backup!

If you don't have a backup, you don't have data. It is not a matter of if a drive will fail, but when. A professional Linux administrator implements a robust, automated backup strategy that can recover from a single lost file or a total system catastrophic failure. This module covers the tools you need to create reliable archives, synchronize directories across servers, and safely dump databases.

1. Core Backup Tools

  • tar (Tape Archive): The standard for creating archives. It bundles multiple files into a single archive file, often compressed with gzip (e.g., .tar.gz).
  • rsync: The ultimate tool for synchronization. It only copies the differences between the source and destination, making it incredibly fast and efficient for backups.
  • Database Dumps: For databases, you need a logical dump (e.g., mysqldump) rather than just copying raw database files, to ensure data consistency.

# Create a compressed backup
tar -czvf backup.tar.gz /path/to/data

# Sync to a remote server
rsync -avz /local/data/ user@backup-server:/remote/backup/

2. Backup Best Practices: The 3-2-1 Rule

A good strategy follows the "3-2-1" rule: Have 3 copies of your data, on 2 different media types, with 1 copy stored off-site. Your local backup won't save you if the building catches fire.

Real-World Relationship: The Fire-Proof Safe

Think of Backups like having a Fire-Proof Safe for your most important documents. If you only keep the originals on your desk, a fire destroys them. If you keep a copy in the safe in the same office, you're better off, but a building fire still destroys both. That is why you must have an off-site copy (the off-site/cloud backup). The 3-2-1 Rule is the insurance policy that guarantees you can always resume business, no matter what happens to the primary hardware.
