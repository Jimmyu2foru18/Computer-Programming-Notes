Linux I: Page 7 - Permissions & Ownership

The Foundation of Linux Security

Linux is inherently a multi-user operating system. This means security is built into the filesystem itself. Every file and directory has an Owner and a Group associated with it, and it has specific Permissions (Read, Write, Execute) that determine who can interact with it. Understanding these concepts is not optional—it is the single most important skill for securing your Linux machine and preventing accidental or malicious damage.

1. The Permission Triad (rwx)

Permissions are defined for three levels of users: User (owner), Group, and Others. Each can have:

  • r (Read): Ability to see file content or list directory files.
  • w (Write): Ability to edit file content or create/delete files in a directory.
  • x (Execute): Ability to run a file as a program or enter a directory.

# Example: -rwxr-xr--
# User: Read, Write, Execute
# Group: Read, Execute
# Others: Read

2. Managing Access

You use chmod to change permissions and chown to change ownership.

Permissions (chmod):

You can use symbolic mode (u+x) or numeric mode (755, where r=4, w=2, x=1).


# Numeric mode (7 = 4+2+1, 5 = 4+0+1)
chmod 755 script.sh # Owner: rwx, Group: r-x, Others: r-x

# Symbolic mode
chmod o-r file.txt # Remove read permission from Others

Ownership (chown):

chown user:group file.txt changes the owner to 'user' and the group to 'group'.

Real-World Relationship: The Office Key Card

Think of File Permissions like Key Cards in an Office Building. The Owner is the employee assigned to a specific office; they have full access (rwx). The Group is the department (e.g., Marketing); they have limited access (r-x) to shared folders. Others are visitors; they might only be allowed to read public documents in the lobby (r--). If you try to enter an office you don't belong to, your key card is rejected (Permission Denied). chmod is simply the security administrator changing your access level, and chown is assigning a different employee to that office.
