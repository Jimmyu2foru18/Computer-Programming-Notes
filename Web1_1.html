Web Technologies I: Page 1 - Introduction to the Web & HTML Structure

The World Wide Web: How it Works

The World Wide Web is a massive system of interconnected documents, images, and applications accessible via the internet. It relies on a Client-Server Architecture: your browser (the client) sends a request (using HTTP/HTTPS) to a remote computer (the server), which processes the request and sends back the content to be rendered on your screen.

The Web Stack

  • HTML (HyperText Markup Language): The structural skeleton of every web page.
  • CSS (Cascading Style Sheets): The visual makeup and layout.
  • JavaScript: The brain—handling interactivity and dynamic behavior.

1. The Foundation: HTML

HTML is not a programming language; it is a Markup Language. It tells the browser what content is on the page (is this a heading? a paragraph? an image?). It uses Tags, which act as containers for content.

Basic Document Structure

Every HTML file follows a standard template:


<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>Main Heading</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

Real-World Relationship: The Book Structure

Think of HTML like the Outline and Text of a book. The `<head>` contains information about the book—the title, author, and edition—which you don't read while reading the story itself. The `<body>` contains the actual chapters, paragraphs, and headings. Just like a book needs a clear outline so the reader can understand the narrative flow, a web page needs clear HTML structure so the browser (and the search engines) can understand the content and hierarchy of your page.
