Web Technologies I: Page 6 - JavaScript: Basics & Variables

The Brain of the Web

HTML is the skeleton, CSS is the makeup, but JavaScript (JS) is the brain. JavaScript is the programming language of the web. It allows you to create dynamic, interactive content, respond to user actions (like clicking buttons or typing in forms), fetch data from servers, and update the UI without reloading the page. Learning JavaScript is the turning point where you stop just "designing" web pages and start building "web applications."

1. Variables: Storing Data

To do anything useful, a program must store information. We do this using Variables. In modern JavaScript (ES6+), you should almost always use let or const.

  • let: Use this when the value of the variable will change over time.
  • const: Use this when the value should never be reassigned. (This is the default—start here!).
  • var: Avoid using this. It has outdated scoping rules that cause bugs.

const API_KEY = "12345"; // Never changes
let score = 0;           // Will change
score = 10;              // Allowed

2. Core Data Types

JavaScript is dynamically typed, meaning a variable can hold any type of data, and that type can change.

  • String: Text (e.g., "Hello World").
  • Number: Integers or floats (e.g., 42, 3.14).
  • Boolean: True or False (true, false).
  • Array: Ordered list (e.g., [1, 2, "three"]).
  • Object: Key-value pairs (e.g., {name: "John", age: 30}).

Real-World Relationship: The Labelled Boxes

Think of Variables like Cardboard Boxes in a Storage Unit. Each box has a Label (the variable name) that tells you what's inside. When you need to use the data later, you just find the box with the correct label. const is a box that you seal with superglue—once you put something in, you can't take it out or replace it. let is a box with a removable lid—you can take out what's inside and replace it with something else whenever you need to. Data Types are just the "type" of thing you put in the box (a "String" item vs. a "Number" item), and JavaScript doesn't care if you put a number in a box that used to hold a string!
