DSA I: Page 6 - Linear Search

The Simple, Sequential Approach

Linear Search (also known as Sequential Search) is the simplest searching algorithm in computer science. It works by checking every single element in a collection one by one, from start to finish, until either the target element is found or the end of the collection is reached. It makes no assumptions about whether the data is sorted, making it incredibly versatile but also potentially very slow for large datasets.

1. The Algorithm

The logic is straightforward:

  1. Start at the first element (index 0).
  2. Compare the current element with the target.
  3. If they match, return the current index (success!).
  4. If they don't match, move to the next element.
  5. If you reach the end of the collection without a match, return -1 (not found).

def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i # Found
    return -1 # Not found

2. Time Complexity

Linear search is the textbook example of O(n) complexity:

  • Worst Case: O(n). The target is at the very end, or not in the array at all, requiring you to check all n elements.
  • Best Case: O(1). The target is the very first element you check.

Real-World Relationship: Searching for a Book on an Unorganized Shelf

Think of Linear Search like searching for a specific book on a completely unorganized bookshelf. You have no idea where the book is, so you have no choice but to look at the first book, check if it's the one you want, look at the second book, check it, and so on. You must look at every single book until you eventually find the right one (or verify it's not there). This is tedious, but if you don't know the system of organization, it's the only guaranteed way to find it.
