Web Technologies I: Page 3 - CSS Fundamentals

Styling the Web

If HTML is the structural skeleton of your website, CSS (Cascading Style Sheets) is the visual makeup, clothing, and styling. CSS allows you to separate content (HTML) from presentation (CSS), which is a fundamental principle of modern web development. By using external CSS files, you can style hundreds of pages simultaneously just by changing one line of code, ensuring consistency and dramatically reducing maintenance time.

1. The Syntax: Rule Sets

A CSS rule set targets an HTML element and tells the browser how to render it. It consists of a Selector and a Declaration Block.


/* Selector */
p {
  /* Declaration Block (Property: Value) */
  color: blue;
  font-size: 16px;
}

Common Selectors:

  • Element: Targets by tag (e.g., h1).
  • Class (.): Targets a group (e.g., .button).
  • ID (#): Targets one unique element (e.g., #main-nav).

2. Ways to Include CSS

You can apply CSS in three ways, but they are not created equal in terms of maintainability:

  1. Inline: Inside the HTML tag using the style attribute (Bad practice—hard to maintain).
  2. Internal: Inside a <style> tag in the <head> (Good for single pages).
  3. External: A separate .css file linked in the <head> (Best practice—use this for 99% of your projects).

<!-- The Best Way: External Link -->
<link rel="stylesheet" href="style.css">

Real-World Relationship: The Interior Decorator

Think of HTML like the Construction of the house—the walls, the floors, the doors. Think of CSS like the Interior Decorator. You don't rebuild the house to change the wall color; you hire a decorator to apply paint, hang pictures, and pick furniture. If you want to change the style of the whole house (the whole website), you don't rebuild the walls—you just update the decoration (the External CSS file). This makes it fast, efficient, and easy to change the "vibe" of your house without structural work.
