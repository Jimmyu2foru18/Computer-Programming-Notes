Java I: Page 7 - Text Manipulation: The String Class & StringBuilder

Strings: More Than Just Text

In almost every Java application, you will spend a significant amount of time working with text. Whether it's processing user names, formatting data for a report, or building a SQL query, Strings are ubiquitous. In Java, unlike the numeric types we saw earlier, a String is not a primitive; it is a Class (an object). This means every string comes with a massive toolkit of built-in methods to help you search, split, and transform text.

The Core Rule: Immutability

One of the most important concepts to master in Java is that Strings are immutable. Once a String object is created in memory, its value can never be changed. When you "modify" a string (e.g., str = str + "!"), Java isn't changing the original object—it's creating a brand-new string in a special area of memory called the String Pool and pointing your variable label to the new location.


String s = "Hello";
s = s.concat(" World"); // "Hello" is abandoned, a NEW "Hello World" is created
Memory Alert: Because strings are immutable, concatenating strings in a loop (e.g., adding 1,000 items to a string) is extremely slow and wasteful. It creates 1,000 separate objects in memory!

Essential String Methods

The String class provides dozens of methods. Here are the "Power Users" you'll use every day:

  • length(): Returns the number of characters.
  • equals(): Used to compare the content of strings. (Never use == for strings!)
  • substring(start, end): Extracts a portion of the text.
  • trim(): Removes leading and trailing whitespace.
  • split(","): Breaks a string into an array based on a delimiter.

String data = " Apple,Banana,Orange ";
String cleaned = data.trim();
String[] fruits = cleaned.split(",");
System.out.println(fruits[0].equals("Apple")); // true

Efficient Building: StringBuilder

When you need to perform heavy text manipulation (like building a large HTML document or processing a file line-by-line), you should use StringBuilder. Unlike String, StringBuilder is mutable. It allows you to modify the content without creating new objects every time.


StringBuilder sb = new StringBuilder();
for (int i = 0; i < 100; i++) {
    sb.append("Item ").append(i).append("\n");
}
String finalResult = sb.toString();

Real-World Relationship: The Book and the Chalkboard

Think of a String like a Printed Book. Once the book is printed, the ink is dry (Immutability). If you want to change one sentence, you have to print a completely new edition of the book. The String Pool is like a Library: if 1,000 people want to read "The Great Gatsby," the library doesn't buy 1,000 copies; it just points all 1,000 people to the same shelf (Memory Efficiency). Think of StringBuilder like a Chalkboard. You can write something, erase it, and add to it as many times as you want without throwing away the board. You only take a photo of the board (toString()) when you're completely finished with your work.
