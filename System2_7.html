System Design II: Page 7 - Integration: Gateways & Queues

Decoupling Components

In a large-scale system, components should be loosely coupled. If Service A needs to call Service B directly, they become brittle—if Service B goes down, Service A fails. We use Message Queues to enable asynchronous, reliable communication, and API Gateways to provide a single, centralized entry point for clients, handling routing, security, and protocol translation.

1. Key Integration Tools

  • Message Queues (RabbitMQ, Kafka): Allow services to communicate by putting messages in a buffer. Service A sends a message and continues its work; Service B reads it whenever it's ready.
  • API Gateway (Kong, AWS API Gateway): The traffic controller. It receives client requests, handles authentication, routes to the correct backend service, and aggregates responses.
