Java I: Page 2 - Variables & Primitive Data Types

The Strict World of Java Variables

In our Python journey, we learned about dynamic typing—the ability to slap a label on any object without worrying about its type. Java is different. Java is a statically and strongly typed language. This means that every variable must be declared with a specific data type before it can be used, and that type cannot change during the life of the variable. While this requires more effort from the developer, it results in code that is much faster and catches many errors before the program even runs.

The 8 Primitive Data Types

Java provides eight "Primitive" data types. These are the most basic building blocks of data and are stored directly in the computer's Stack memory for high performance.

Type Size Range / Value Example
byte 8-bit -128 to 127 byte age = 25;
short 16-bit -32,768 to 32,767 short year = 2024;
int 32-bit -2³¹ to 2³¹-1 int count = 1000;
long 64-bit Huge whole numbers long pop = 8000000L;
float 32-bit Decimals (low prec) float price = 1.99f;
double 64-bit Decimals (high prec) double pi = 3.14;
boolean 1-bit true / false boolean ok = true;
char 16-bit Unicode character char grade = 'A';
Pro Tip: Always use long for large integers and double for decimals unless you have a specific reason to save memory with the smaller types. Note the 'L' suffix for longs and 'f' for floats!

Declaration, Initialization, and Constants

In Java, you "declare" a variable to tell the compiler to set aside memory. You "initialize" it to give it a starting value. You can do both in one line.


int score;              // Declaration
score = 95;             // Initialization

final double PI = 3.14; // A Constant (cannot be changed)

Variable Scope

The Scope of a variable determines where in your code that variable is "visible" and can be used. Java has several levels of scope:

  • Local Variables: Defined inside a method. They only exist while the method is running.
  • Instance Variables: Defined in a class but outside methods. Each object has its own copy.
  • Static (Class) Variables: Shared by all instances of the class.

Real-World Relationship: The Toolbox and the Custom Mold

Think of Python Variables like a Magic Bag. You can throw anything inside—a ball, a book, a sandwich—and the bag just expands to fit it. Java Variables are like a Set of Specific Containers. If you want to store a liquid, you must use a bottle. if you want to store a sandwich, you must use a box. You can't put a sandwich in the bottle. This strictness (Strong Typing) ensures that you never accidentally try to pour a sandwich or eat a bottle. Primitives are like the Tools in your pocket—you carry them everywhere for instant use. Reference Types (which we'll cover later) are like a Claim Ticket for a large item in a warehouse—the variable doesn't hold the item itself, just the address where the JVM can find it.
