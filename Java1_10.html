Java I: Page 10 - Deep Dive: Constructors & the Static Keyword

The Birth of an Object

In our previous page, we learned that the new keyword creates an object instance from a class. But what happens at that exact moment of creation? How does the object get its initial data? This is the job of the Constructor. A constructor is a special "method" that has the same name as the class and no return type. Its sole purpose is to initialize the state of a new object. Additionally, we will explore the static keyword, which allows us to manage data at the Class level rather than the individual object level.

1. Master of Initialization: Constructors

If you don't write a constructor, Java provides a "Default Constructor" (with no parameters) that does nothing. However, in professional code, we usually write our own to ensure objects are valid the moment they are created.

Constructor Overloading

Just like methods, you can have multiple constructors with different parameters. This gives you flexibility in how you create objects.


public class User {
    String name;
    int level;

    // Parameterized Constructor
    public User(String name, int level) {
        this.name = name;
        this.level = level;
    }

    // Overloaded Constructor (Default level to 1)
    public User(String name) {
        this(name, 1); // Calls the other constructor!
    }
}

2. Class-Level Logic: The static Keyword

By default, attributes and methods belong to an Instance (each object has its own copy). When you use static, the member now belongs to the Class itself. There is only one copy in memory, no matter how many objects you create.

  • Static Variables: Often used for counters or constants (e.g., tracking total users).
  • Static Methods: Often used for utility functions that don't need object data (e.g., Math.sqrt()).

public class MathUtils {
    public static final double PI = 3.14159; // Global constant

    public static int square(int n) {
        return n * n;
    }
}
// Called using the Class name, NOT an object:
int result = MathUtils.square(5);

Real-World Relationship: The Birth Certificate and the Public Park

Think of a Constructor like a Birth Certificate. When a new person is born, the hospital doesn't just create a blank human; they immediately record the name, parentage, and time of birth. Without this "initialization," the human wouldn't have a valid identity in the system. Think of Static Variables like a Public Park in a neighborhood. Every house (Object) in the neighborhood has its own private backyard (Instance variables), but there is only one park (Static variable) that everyone shares. If one person plants a tree in the park, everyone else sees it. If the neighborhood grows to 1,000 houses, there is still only one park. This is the ultimate tool for sharing data efficiently without wasting memory on 1,000 copies of the same information.
