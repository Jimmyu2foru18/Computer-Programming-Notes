Networks II: Page 5 - The Web: HTTP & Secure Protocols

The Language of the Web

Almost everything you do in a web browser is governed by the Hypertext Transfer Protocol (HTTP). HTTP is a request-response protocol—your browser (the Client) sends a request to a website (the Server), and the server sends back the web page content as a response. In the modern, secure web, we almost exclusively use HTTPS, which adds a layer of encryption (TLS/SSL) to HTTP, ensuring that nobody can spy on your traffic or steal your passwords. Understanding how HTTP requests and responses work is fundamental to web development and API design.

1. The Request-Response Cycle

Every web action follows a strict lifecycle. A request consists of a method, a URL, and headers. A response consists of a status code and the actual page content (HTML/JSON/etc.).

Common Methods:

  • GET: Request data (e.g., loading a page).
  • POST: Submit data (e.g., logging in, submitting a form).
  • PUT/PATCH: Update existing data.
  • DELETE: Remove data.

Status Codes (The "Voice" of the Server):

  • 2xx (Success): 200 OK, 201 Created.
  • 4xx (Client Error): 404 Not Found, 401 Unauthorized.
  • 5xx (Server Error): 500 Internal Server Error.

2. HTTPS: Encryption and Security

HTTP is "cleartext"—if you send a password over HTTP, anyone on the same public Wi-Fi can see it. HTTPS solves this by using TLS (Transport Layer Security), which encrypts the entire HTTP conversation. Before any data is sent, the client and server perform a "Handshake" to verify certificates and exchange encryption keys.

Real-World Relationship: The Restaurant Waiter

Think of HTTP like a Waiter at a Restaurant. You (the Client/Browser) give the waiter an order (the Request). The waiter takes it to the kitchen (the Server). The kitchen prepares the dish and hands it back to the waiter, who delivers it to your table (the Response). If you order something that isn't on the menu, the waiter returns with a "Sorry, that's not available" (404 Error). Think of HTTPS like a Waiter wearing a Mask and talking to you in Code. Nobody else in the restaurant can hear what you're ordering, and you have proof that the waiter actually works for the restaurant (the SSL Certificate) rather than being an imposter trying to steal your order.
