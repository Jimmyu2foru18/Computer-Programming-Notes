Networks I: Page 7 - Practical Networking: The TCP/IP Model

Theory vs. Practice

In the last two pages, we explored the OSI Model. It's a wonderful theoretical framework for understanding the concept of how networks work. However, the internet wasn't built on the OSI model; it was built on the TCP/IP Model. The TCP/IP model is a more practical, implementation-oriented framework that maps the complex seven layers of OSI into four more streamlined, functional layers. Understanding both models is crucial: OSI gives you the academic foundation, while TCP/IP gives you the tools to build and troubleshoot the actual internet.

1. The Four Layers of TCP/IP

Layer OSI Equivalent Key Function
Application 5, 6, 7 User applications, protocols like HTTP/FTP
Transport 4 End-to-end communication (TCP/UDP)
Internet 3 Routing and logical addressing (IP)
Network Access 1, 2 Physical hardware, framing (Ethernet/Wi-Fi)

2. Data Encapsulation: The "Package" Process

As data travels down the stack, each layer wraps the data with its own header, creating a new "package" for transmission. This process is called Encapsulation. When the data reaches its destination, the process is reversed (Decapsulation).

Real-World Relationship: The Shipping Logistics

Think of Encapsulation like Shipping a Package. You have your item (the Data). You put it in a small box (Transport Layer/Segment). You put that box inside a larger shipping container (Internet Layer/Packet) with a specific destination address. Finally, the container is loaded onto a specific truck or train (Network Access/Frame) to be physically transported. Each stage of the journey only cares about the packaging that was added by the person before them. They don't need to know what's inside the box—they just need to know how to move the container to the next hub (the Router) until it reaches the final destination.
