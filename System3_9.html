System Design III: Page 9 - Advanced Architecture Project

The Capstone Challenge

You've learned the individual components: Frontend components, Backend APIs, SQL/NoSQL databases, Security, CI/CD, and Scalability. Now, it's time to synthesize everything. The goal of this capstone is to design a complex, scalable system—like a Global URL Shortener (e.g., bit.ly). This is a classic interview question that requires you to apply every single concept we've covered to handle massive traffic and data storage.

1. The Challenge Requirements

  • Functional: Shorten a long URL; redirect users to the original URL.
  • Non-Functional: Handle 10,000 requests per second (RPS); <100ms latency; 99.99% availability.

Architectural Components:

  • Frontend: Simple form to input URL, display shortened link.
  • Backend API: Express.js or FastAPI (highly concurrent).
  • Database: Needs massive scale for read-heavy operations (KV Store like Redis or Cassandra).
  • Caching: Put Redis in front of the DB to avoid hits for popular URLs.
