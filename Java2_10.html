Java II: Page 10 - Advanced Topics: Collections & Generics

Dynamic Data and Type Safety

In our earlier module, we learned about Arrays. While arrays are fast, their fixed-size nature makes them difficult to use for dynamic data (like a list of items in a shopping cart that can grow or shrink). To solve this, Java provides the Collections Framework—a set of classes and interfaces that implement commonly used data structures like dynamic lists, sets, and maps. To make these collections safe and easy to use, Java also uses Generics, which allow you to specify exactly what type of data a collection can hold.

1. The Collections Framework

The framework is organized into three main interfaces:

  • List: An ordered collection that allows duplicates (e.g., ArrayList).
  • Set: An unordered collection of unique items (e.g., HashSet).
  • Map: A collection of Key-Value Pairs (e.g., HashMap).

// A dynamic list of Strings
List names = new ArrayList<>();
names.add("Alice");
names.add("Bob");

// A unique set of Integers
Set uniqueIds = new HashSet<>();
uniqueIds.add(101);
uniqueIds.add(101); // Automatically ignored!

// A map of student names to their grades
Map grades = new HashMap<>();
grades.put("Alice", 95.5);

2. Understanding Generics (<T>)

Before Java 5, collections could hold any Object, which often led to dangerous runtime errors. Generics allow you to enforce Type Safety at compile-time. When you write ArrayList<String>, you are telling the compiler: "This list can ONLY hold Strings." If you try to add an Integer, the code won't even compile.

3. Iterating with Streams (Java 8+)

Modern Java allows you to process collections using Streams, which provide a functional way to filter, map, and reduce data.


names.stream()
     .filter(n -> n.startsWith("A"))
     .forEach(System.out::println);

Real-World Relationship: The Accordion and the Safe

Think of an ArrayList like an Accordion. Unlike a piano which has a fixed number of keys (an Array), an accordion can expand and contract as needed to accommodate more data. Think of Generics like a High-Security Safe with a Shape-Coded Lock. If the safe is designed for "Gold Bars" (Strings), and you try to shove in a "Lead Weight" (an Integer), the lock simply won't turn. This prevents you from accidentally mixing up your assets (Data Types) and discovering the mistake only when it's too late. Finally, think of HashMap like a Coat Check at a theater: you give them your coat (the Value) and they give you a unique ticket (the Key). To get your coat back, you don't search through 1,000 coats; you just hand them the ticket, and they walk straight to your item.
