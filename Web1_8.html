Web Technologies I: Page 8 - JavaScript: Async & Fetch

Handling the Wait

In web development, you frequently need to do things that take time—fetching data from a server, reading a large file, or waiting for a timer. If JavaScript were strictly "synchronous" (doing one thing at a time, blocking everything else), the entire browser would freeze every time you made a network request. Asynchronous JavaScript allows you to start an operation and continue doing other things, only responding once the operation finishes. Promises and Async/Await are the modern tools used to manage these background tasks effectively.


setTimeout(() => {
  console.log("Done waiting!");
}, 2000);

1. Promises: I Promise to Finish

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation. It can be in one of three states:

  • Pending: The operation is still running.
  • Resolved (Fulfilled): The operation succeeded.
  • Rejected: The operation failed.

const myPromise = new Promise((resolve, reject) => {
  // Simulate network request
  setTimeout(() => resolve("Data loaded!"), 2000);
});

myPromise.then(data => console.log(data));

2. Fetch API: Talking to Servers

The Fetch API is the modern standard for making network requests (e.g., getting data from an API). It returns a Promise, making it incredibly easy to use with async/await syntax.


async function loadUserData() {
  const response = await fetch("https://api.example.com/user/1");
  const data = await response.json();
  console.log(data);
}

// Practice: A simple async delay
const delay = (ms) => new Promise(resolve => setTimeout(resolve, ms));
delay(1000).then(() => console.log("Waited 1 second"));

Real-World Relationship: The Restaurant Pager

Think of Asynchronous JavaScript like a Restaurant Pager. You place your order (Start Request), and instead of standing at the counter blocking everyone else (Synchronous Blocking), you take a pager and go sit down to have a conversation (Continue Doing Other Work). When the kitchen finishes your order (Async Operation Completes), your pager goes off (Promise Resolves). You then walk up to the counter and collect your food. This allows the restaurant (the Browser) to serve hundreds of customers simultaneously without a massive bottleneck at the order counter.


// The "Pager" in code
const orderFood = () => {
  console.log("Order placed...");
  return new Promise(resolve => setTimeout(() => resolve("Pizza!"), 3000));
};

orderFood().then(food => console.log(`Got my ${food}`));
