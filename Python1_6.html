Python I: Page 6 - Collection Types: Tuples & Sets

The Power of Constraints: Tuples (tuple)

In our previous page, we learned that Lists are mutable—you can change them anytime. But what if you have data that should not change? For example, the latitude and longitude of a city, or the RGB values of a specific color. For these scenarios, Python provides the Tuple. A tuple is an ordered collection of items that is immutable. Once a tuple is created, it cannot be modified, which makes it safer and faster than a list.

Creating Tuples

Tuples are defined using parentheses (). A common trap for beginners is creating a tuple with only one item—you must include a trailing comma, or Python will treat it as a regular variable in parentheses.


coordinates = (10, 20)
colors = ("red", "green", "blue")
single_item = (42,)  # Note the comma! Without it, this is just the number 42

Tuple Unpacking: A Professional Shortcut

Python allows you to "unpack" the contents of a tuple directly into variables in a single line. This is a very common pattern when functions return multiple values.


x, y = coordinates
print(f"X: {x}, Y: {y}")

Unique Collections: Sets (set)

Imagine you have a list of 10,000 email addresses and you want to know how many unique users you have. A List would make you check every item manually. A Set, however, is an unordered collection of unique items. It automatically removes any duplicates you try to add to it.

Set Basics

Sets are defined using curly braces {} (similar to dictionaries, but without key-value pairs) or the set() function.


# Automatic duplicate removal
numbers = {1, 2, 2, 3, 3, 4}
print(numbers)  # Output: {1, 2, 3, 4}

fruits = {"apple", "banana"}
fruits.add("orange")

Set Mathematics: Union, Intersection, and Difference

Sets in Python are modeled after mathematical sets, allowing you to perform powerful comparisons between two groups of data instantly.

  • Union (|): Combines all items from both sets (no duplicates).
  • Intersection (&): Returns only the items found in both sets.
  • Difference (-): Returns items in the first set that are not in the second.

set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

print(set1 | set2)  # {1, 2, 3, 4, 5, 6} (Union)
print(set1 & set2)  # {3, 4} (Intersection)
print(set1 - set2)  # {1, 2} (Difference)

Real-World Relationship: The GPS Coordinate and the Guest List

Think of a Tuple as a GPS Coordinate printed on a stone tablet. You can read the North and East values (Unpacking), but you can't just chip away at the stone to change the numbers—if you want a new location, you need a new tablet (Immutability). Think of a Set as a VIP Guest List at a club. The bouncer (Python) checks every person trying to enter. If "Alice" is already in the club, he won't let another "Alice" in (Uniqueness). If you want to know which guests are both "VIPs" and "Early Birds," you look for the Intersection of those two lists. This ensures your data remains clean, unique, and organized without manual sorting.
