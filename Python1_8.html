Python I: Page 8 - Control Flow: Conditionals & Logic

The Brain of Your Program

In our previous pages, we learned how to store and organize data. Now, we need to learn how to make that data useful. Control flow is what allows a program to make decisions, execute different blocks of code based on conditions, and respond dynamically to user input. The most fundamental tool for this is the Conditional Statement.

Decision Branches: if, elif, and else

Python uses a clean, indentation-based syntax to define logic branches. The if statement checks a condition; if it is true, the code inside the block runs. If not, Python moves to the elif (else if) blocks, and finally to the else block if no other conditions were met.


score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

print(f"Your grade is: {grade}")

The Ternary Operator: One-Line Logic

Sometimes, an entire if-else block is overkill for a simple assignment. Python provides a "ternary operator" (or conditional expression) that allows you to make a decision in a single line. It follows the pattern: value_if_true if condition else value_if_false.


status = "Pass" if score >= 60 else "Fail"
print(status)  # Output: Pass

Logical Operators and Precedence

You can combine multiple conditions using and, or, and not. However, you must be careful with Precedence—the order in which Python evaluates these operators. not is evaluated first, then and, and finally or. When in doubt, use parentheses () to make your logic explicit.


# Complex logic
if (is_weekend or is_holiday) and not is_working:
    print("Time to relax!")

Real-World Relationship: The Smart Thermostat and the Security Gate

Think of Conditionals like a Smart Thermostat. It's constantly checking: if the temperature is below 68, turn on the heat. elif it's above 75, turn on the AC. else, do nothing. It makes real-time decisions based on sensor data. Think of the Ternary Operator like a Security Gate at a parking garage. There's no complex decision tree needed—it's a binary choice: status = "Open" if ticket_valid else "Closed". Finally, Logical Precedence is like a Contract: the fine print (parentheses) and the specific order of clauses (and/or) determine whether the deal is valid or not. Misunderstanding one "and" could change the entire meaning of the agreement.
