Linux I: Page 5 - File System Hierarchy

Everything is a File

In Linux, there is a fundamental design principle: "Everything is a file." Not just your documents and photos, but also your keyboard, your hard drives, your network interface, and even the processes running in memory. To manage this massive amount of data, Linux follows the File System Hierarchy Standard (FHS). This standard ensures that whether you're using Ubuntu, Fedora, or a server distro, you'll always find configuration files in /etc, user data in /home, and temporary files in /tmp. Understanding this structure is essential for locating system tools and configuration files.

1. Mapping the Hierarchy

The hierarchy starts at the Root directory, denoted by a single slash (/). From there, it branches into specialized directories:

  • /bin & /sbin: Essential executable programs (e.g., ls, cp).
  • /etc: System-wide configuration files (the heart of system admin).
  • /home: User-specific files (documents, downloads).
  • /root: The home directory of the root (administrator) user.
  • /var: Variable data, like system logs (/var/log) and databases.
  • /proc & /sys: Virtual filesystems that represent system processes and hardware state.

2. Why FHS Matters

Without the FHS, every developer would put their files wherever they wanted. The system would become unmanageable. The FHS provides a predictable "map," allowing administrators to quickly find what they need to fix, backup, or configure, regardless of who installed the software.

Real-World Relationship: The City Hall & Filing System

Think of the File System Hierarchy like a Well-Organized City. The Root (/) is the City Hall, where all addresses originate. /etc is the Office of Records, where all city ordinances and configuration rules are kept. /home is a residential neighborhood where families keep their personal property. /var is the City Archive, where they keep logs of every visitor and transaction that happened that day. If a city didn't have these organized districts, the police (system tools) wouldn't know where to look to keep things running, and the citizens (users) would be hopelessly lost.
