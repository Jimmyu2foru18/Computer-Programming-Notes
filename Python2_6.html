Python II: Page 6 - Object-Oriented Programming (OOP): Inheritance & MRO

The Power of Reusability

In the physical world, we rarely build everything from scratch. When a company designs a new smartphone, they don't reinvent the concept of a "battery" or a "screen"; they take existing technology and build on top of it. In programming, we do the same thing using Inheritance. Inheritance allows us to create a new class (the Child or Derived class) that inherits all the attributes and methods of an existing class (the Parent or Base class). This promotes code reuse and establishes a logical hierarchy in your software.

Base vs. Derived Classes

To make one class inherit from another, you simply put the parent class's name in parentheses after the child class's name. The child class can then add its own methods or override methods from the parent to provide specialized behavior.

Using super()

The super() function is a built-in tool that allows you to call methods from the parent class inside the child class. It is most commonly used in the __init__ method to ensure the parent's data is initialized correctly.


class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return "Generic sound"

class Dog(Animal):
    def __init__(self, name, breed):
        # Initialize the Parent's data first
        super().__init__(name)
        self.breed = breed
    def speak(self):
        # Override the Parent's method
        return "Woof!"

The Complexity of Multiple Inheritance

Python is one of the few modern languages that supports Multiple Inheritance, where a single class can have multiple parents. While powerful, this can lead to conflicts if two parents have a method with the same name. This is known as the "Diamond Problem."

Inheritance Simulator

Method Resolution Order (MRO)

Python solves inheritance conflicts using an algorithm called C3 Linearization to create a Method Resolution Order (MRO). This is the exact order in which Python searches for a method in the hierarchy. You can check any class's MRO by calling the .mro() method.


print(Dog.mro())

Real-World Relationship: Evolution and Smartphone Generations

Think of Inheritance like Biological Evolution. Every animal inherits a set of basic blueprints from its ancestors (DNA). A wolf inherits the ability to breathe and eat from the general category of "Mammals." Over time, the "Dog" class diverged, inheriting the wolf's structure but "overriding" behaviors to be more friendly to humans. Think of super() like a Family Heritage: you carry the traditions of your parents while adding your own unique personality. Finally, think of MRO like a Legal Chain of Command. If there is a dispute (a method conflict) in a multi-national company, the MRO is the official rulebook that decides which country's law (the parent class) takes priority based on the contract hierarchy.
