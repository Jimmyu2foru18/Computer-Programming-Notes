System Design II: Page 3 - Backend: GraphQL vs REST

Evolving API Design

While REST is the long-standing standard, GraphQL has emerged as a powerful alternative. The fundamental problem with REST is "over-fetching" (receiving more data than you need) or "under-fetching" (needing to make multiple requests to get related data). GraphQL solves this by allowing the client to define the exact data structure it needs, and the server returns exactly that, in a single query.

1. Comparison

  • REST: Resource-based. Multiple endpoints. Fixed response structures. Easier to cache.
  • GraphQL: Query-based. Single endpoint. Flexible response structures. Harder to cache.

# GraphQL Query
query {
  user(id: "1") {
    name
    posts { title }
  }
}
