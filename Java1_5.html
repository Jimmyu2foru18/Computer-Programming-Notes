Java I: Page 5 - Control Flow: Iteration with Loops

The Power of Repetition

In programming, we rarely perform an action just once. We process thousands of user records, update every pixel on a screen, or wait for a network connection to respond. Doing this manually would be impossible. Loops allow us to write a block of code once and tell the JVM to repeat it as many times as necessary. Java provides four distinct types of loops, each designed for specific iteration scenarios.

1. The for Loop: Definite Iteration

The traditional for loop is used when you know exactly how many times you want the code to run. It consists of three parts: initialization, condition, and increment/decrement.


for (int i = 0; i < 5; i++) {
    System.out.println("Iteration number: " + i);
}
Expert Note: The scope of the counter variable i is limited to the loop itself. This is a clean way to manage temporary variables without cluttering the rest of your method.

2. The while and do-while Loops

These loops are used when you don't know the end point in advance. They repeat as long as a condition remains true.

The while Loop

Checks the condition before executing the block. If the condition is false initially, the loop never runs.

The do-while Loop

Executes the block first, then checks the condition. This guarantees the code will run at least once.


int count = 0;
do {
    System.out.println("This runs once no matter what!");
    count++;
} while (count < 0); // Condition is false, but it already ran once.

3. Loop Control: break and continue

Sometimes you need to change the flow of a loop from the inside.

  • break: Immediately exits the loop entirely.
  • continue: Skips the rest of the current iteration and jumps straight to the next check.

for (int i = 0; i < 10; i++) {
    if (i == 3) continue; // Skip 3
    if (i == 7) break;    // Stop at 7
    System.out.println(i);
}

Real-World Relationship: The Assembly Line and the Loading Screen

Think of a For Loop like an Industrial Assembly Line. You have 500 cars on a conveyor belt. The robot arm knows it has to perform the same task exactly 500 times before the shift is over. Think of a While Loop like a Loading Screen in a modern video game. The screen keeps spinning while (dataLoading == true). The computer doesn't know if your internet is fast or slow, so it just keeps looping until the data is ready. Finally, think of a Do-While Loop like Entering a Password. You must enter the password at least once before the computer can check if it's correct. You "do" the input, then "while" it's wrong, you repeat.
