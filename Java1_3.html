Java I: Page 3 - Operators & Boolean Logic

The Engine of Calculation

In the previous page, we learned how to store data. Now, we must learn how to manipulate it. Operators are special symbols in Java that perform specific operations on one, two, or three operands and return a result. From basic math to complex logical checks, operators are the engine that drives your program's logic. Java's operators are highly optimized and follow strict rules of precedence, similar to the order of operations you learned in algebra.

1. Arithmetic Operators

Java supports the standard mathematical operators you would expect, along with some powerful shortcuts for modifying variables.

  • +, -, *, /: Standard math.
  • %: Modulo (returns the remainder of division).
  • ++ / --: Increment and Decrement. These are used to increase or decrease a variable's value by 1.

int x = 10;
int y = 3;
System.out.println(x % y);  // Output: 1 (Remainder)

x++; // x is now 11
y--; // y is now 2
Expert Note: In Java, 10 / 3 will return 3, not 3.33. This is because both operands are integers, so Java performs Integer Division. To get a decimal, at least one number must be a double: 10.0 / 3.

2. Comparison & Logical Operators

Comparison operators compare two values and return a boolean (true or false). Logical operators are used to combine multiple boolean expressions.

Short-Circuit Evaluation

Java uses Short-Circuit operators (&& and ||). This means that if the result of a logic chain can be determined by the first part, Java won't even look at the second part. This is highly efficient and prevents many errors.


int age = 25;
boolean hasLicense = true;

// Both must be true
if (age >= 18 && hasLicense) {
    System.out.println("You can drive!");
}

// Only one needs to be true
if (age < 18 || !hasLicense) {
    System.out.println("You cannot drive alone.");
}

Operator Precedence

Like PEMDAS in math, Java evaluates operators in a specific order. Postfix operators (x++) have the highest priority, followed by multiplicative (*, /, %), additive (+, -), comparison, and finally logical operators. When in doubt, always use parentheses to make your intent clear to other developers.

Real-World Relationship: The Calculator and the Security Guard

Think of Arithmetic Operators like the Mechanical Gears in a physical calculator. Every rotation (operation) produces a predictable, physical result. Think of Logical Operators like a Security Guard at a gate. The guard has a checklist: (Has Ticket OR Is Employee) AND (Is Not Banned). The Short-Circuit (&&) is like the guard stopping the moment they see you don't have a ticket—they don't even bother checking if you're banned because the result is already "No Entry." This efficiency is what allows Java applications to process millions of transactions per second without wasting a single CPU cycle.
