Web Technologies I: Page 2 - HTML Elements & Semantics

The Vocabulary of the Web

An HTML document is more than just a template; it is a collection of elements that define the document's content and meaning. If you think of HTML as the skeleton of a website, then HTML Elements are the individual bones, joints, and organs. Knowing which tag to use for which piece of content is the difference between a website that is just "visual" and a website that is robust, accessible, and searchable.

1. The Essential Elements

While there are dozens of HTML tags, you will use a core set daily. Understanding their purpose is critical for proper structure.

  • Headings (<h1>-<h6>): Define hierarchy. <h1> is the page title; <h2> are main section titles.
  • Lists (<ul>, <ol>, <li>): Used to organize related pieces of information.
  • Tables (<table>, <tr>, <td>): Used for tabular data (not for page layout!).
  • Forms (<form>, <input>, <button>): The primary way to collect user data.

2. Semantic HTML: The Meaning Behind the Tag

In the early days, everything was a <div>. Today, we use Semantic Elements to tell the browser and search engines what the content is, not just how it looks. This is vital for accessibility (screen readers need to know where the navigation is versus the main content) and SEO (Google needs to know which part of the page is the core topic).

Key Semantic Tags:

  • <nav>: Navigation links.
  • <header>: The top section (logo, branding).
  • <main>: The core content of the page.
  • <article>: Self-contained content (like a blog post).
  • <aside>: Tangential content (sidebars).
  • <footer>: The bottom section (copyright, contact info).

Real-World Relationship: The Architect and the Building Labels

Think of Semantic HTML like Architectural Blueprints. An architect doesn't just draw a bunch of identical squares (<div>s). They label one square "Kitchen," another "Bedroom," and another "Garage." If a contractor walks in, they know exactly what each room is for. If you don't label your rooms (if you only use <div> tags), a visually impaired person (using a screen reader) or a search engine crawler (Googlebot) walks into your "web-house" and has no idea which part of the house is the kitchen or which part is the bathroom. Labels make your "house" livable and functional for everyone.
