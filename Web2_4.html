Web Technologies II: Page 4 - Database Integration: MySQL & PHP

The Heart of the Application

Most modern web applications are essentially "interfaces for databases." Whether it's a blog, an e-commerce site, or a social network, your web application exists to save, retrieve, and modify information stored in a database. MySQL is the most common relational database used with PHP. To build robust applications, you must master CRUD operations: Create, Read, Update, and Delete. Furthermore, you must learn to do this securely, protecting your data from common vulnerabilities like SQL Injection.

1. CRUD Operations

Every database interaction boils down to these four operations. In PHP, we use PDO prepared statements for all of them.

  • Create: INSERT INTO users (name, email) VALUES (?, ?)
  • Read: SELECT * FROM users WHERE id = ?
  • Update: UPDATE users SET name = ? WHERE id = ?
  • Delete: DELETE FROM users WHERE id = ?

// Reading data safely
$stmt = $db->prepare("SELECT name FROM users WHERE id = :id");
$stmt->execute(['id' => $userId]);
$user = $stmt->fetch();

2. Security: Preventing SQL Injection

SQL Injection occurs when a malicious user inputs SQL commands into a form field, potentially allowing them to dump your entire database. Never insert user input directly into a SQL query string. Always use Prepared Statements, which treat user input as data, not as part of the command to be executed.

Real-World Relationship: The Secure Bank Teller

Think of your Database as a Bank Vault and your PHP application as the Bank Teller. The teller (PHP) is the only one who has access to the vault. When a customer (the user) wants to withdraw money (read/modify data), they don't walk into the vault themselves—that would be chaotic and insecure. They tell the teller what they want, and the teller performs the action securely on their behalf. SQL Injection is like a criminal trying to trick the teller by using a fake, forged withdrawal slip to steal money from the vault. Prepared Statements are the "official security-verified withdrawal slips" that make it impossible for a criminal to trick the teller—the teller only accepts official, pre-approved paperwork.
