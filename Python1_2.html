Python I: Page 2 - Variables, Assignment & Naming Conventions

Variables: Storing Your Data

In the world of programming, variables are your most basic building blocks. They are names that refer to values stored in your computer's memory. As we touched on in the previous curriculum, Python is dynamically typed. This is a massive advantage: it means you don't need to tell Python that a variable is a number or a piece of text—it figures it out automatically based on the value you provide.

Basic Assignment

You assign a value to a variable name using the equals sign (=). Think of this as "binding" a name to an object.


name = "Alice"         # String (Text)
age = 25               # Integer (Whole Number)
height = 5.6           # Float (Decimal)
is_student = True      # Boolean (True/False)

Advanced Assignment Techniques

Python allows for some very efficient ways to assign variables that can save you several lines of code.

1. Multiple Assignment

You can assign multiple variables to multiple values in a single line. This is incredibly useful for coordinates or settings.


x, y, z = 1, 2, 3
print(x) # 1
print(y) # 2

2. Chained Assignment

You can also assign the same value to multiple variable names at once.


a = b = c = 0  # All three now equal 0

Naming Conventions: Writing "Clean" Code

While Python gives you freedom in how you name your variables, the professional community follows a set of rules called PEP 8. Following these ensures that your code is readable by other developers around the world.

Good Naming Practices:

  • snake_case: Use lowercase words separated by underscores for variables and functions (e.g., user_name, total_count).
  • Constants: If a value is meant to stay the same forever, use UPPERCASE (e.g., MAX_SIZE = 1000, PI = 3.14).
  • Descriptive: student_list is always better than s or l.

Names to Avoid:

  • Start with Numbers: 1name is invalid and will cause an error.
  • Reserved Keywords: Don't use names like class, if, or print, as Python already uses these for the language itself.
  • Ambiguity: Avoid using lowercase 'L', uppercase 'O', or uppercase 'I' as single-character names, as they can be confused with numbers 1 and 0.

Real-World Relationship: The Filing Cabinet and the Label Maker

Think of your computer's memory as a massive Filing Cabinet. Variables are not the files themselves, but the Labels you put on the folders. Multiple Assignment is like having one folder that holds several related documents, and you put a label on each document simultaneously. Naming Conventions are like the system you use to organize your cabinet—if you label every folder "X" or "Data," you'll never find anything. If you use a consistent system (like "Snake Case" for files and "Uppercase" for the cabinet's master rules), anyone—including your future self—can walk in and understand exactly where everything is stored.
