DSA II: Page 10 - Optimization Problems: Greedy vs. Backtracking

Finding the Best Solution

Many complex problems ask for an "optimal" solution among a sea of possibilities. When the problem is too large to check every single possibility (brute force), we rely on sophisticated strategies. Greedy Algorithms make the best local choice at each step, hoping it leads to a global optimum. Backtracking systematically searches through all possible solutions, "undoing" choices that lead to dead ends. Knowing when to use which is the hallmark of an experienced algorithm engineer.

1. Greedy Algorithms

A Greedy algorithm follows the heuristic of making the locally optimal choice at each stage. It's fast and simple, but it does not always produce the global optimum.

  • Example: Dijkstra’s Algorithm for shortest path (works for non-negative weights).
  • Trade-off: Speed over guaranteed global optimality.

# Example: Greedy approach for Change-making problem
def make_change(amount, coins):
    coins.sort(reverse=True)
    change = []
    for coin in coins:
        while amount >= coin:
            amount -= coin
            change.append(coin)
    return change

2. Backtracking

When you need to explore a search space to find *all* solutions or the *guaranteed* best solution, use Backtracking. It's essentially a depth-first search of the state space. If you reach a state that violates constraints, you "backtrack" to the previous state and try a different path.

Use Cases: N-Queens problem, Sudoku solver, Graph coloring.

Real-World Relationship: The Mountain Hiker

Think of a Greedy Algorithm like a Hiker trying to reach the highest mountain peak by only walking uphill. They will reach a peak very quickly, but it might just be a small hill (local optimum), not the true highest summit (global optimum). Think of Backtracking like a Explorer mapping an entire cave system. They explore one tunnel, find a dead end, come all the way back to the last junction (Backtrack), and explore a different tunnel. They will eventually map the entire cave and find the absolute deepest point, but it takes much, much longer than just walking uphill.
