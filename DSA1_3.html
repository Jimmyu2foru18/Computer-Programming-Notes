DSA I: Page 3 - Linked Lists

The Flexible Chain

While arrays require contiguous memory, Linked Lists store data in nodes scattered throughout memory, connected by pointers. Each node contains two pieces of information: the data itself and a reference (or pointer) to the next node in the sequence. This structure makes Linked Lists the perfect solution when you need to perform frequent insertions and deletions, because you don't need to "shift" subsequent elements as you do with arrays. You simply update the pointer of the previous node to point to the new node!

1. Node Structure & Types

A Linked List is a series of nodes linked together.

  • Singly Linked List: Each node points only to the next node. You can only traverse in one direction.
  • Doubly Linked List: Each node points to both the next and the previous node, allowing bidirectional traversal.

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None # Pointer to next node

2. Complexity Comparison

Linked Lists solve the insertion problem, but at a cost: random access is slow (O(n) because you must traverse nodes).

Operation Linked List Array
AccessO(n)O(1)
Insert (known location)O(1)O(n)
Delete (known location)O(1)O(n)

Real-World Relationship: The Scavenger Hunt

Think of an Array like a Row of Lockers in a hall. You know exactly where everything is. Think of a Linked List like a Scavenger Hunt. You have a clue (data) in your hand, and that clue tells you exactly where to find the *next* clue (the pointer). You can't just skip to the 5th clue without visiting the 1st, 2nd, 3rd, and 4th clues first. But if you want to add a new clue to the middle of the hunt, you don't have to move all the other clues. You just change what the 3rd clue says to make it point to your new 4th clue, which then points to the old 4th clue.
