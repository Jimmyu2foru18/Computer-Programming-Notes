Reusability and Dynamic Dispatch

Inheritance allows you to create specialized classes from general ones, while Polymorphism (via virtual functions) allows you to treat different types through a common interface.

The Code implementation


/*
 * File: inheritance.cpp
 * Demonstration of polymorphism and dynamic dispatch
 */
#include <iostream>
#include <vector>

using namespace std;

// Base class
class Animal {
public:
    // Virtual destructor is crucial for base classes
    virtual ~Animal() {}
    
    // Virtual function enables polymorphism
    virtual void speak() const {
        cout << "Some animal sound" << endl;
    }
};

// Derived classes
class Dog : public Animal {
public:
    void speak() const override { // Explicit override
        cout << "Woof!" << endl;
    }
};

class Cat : public Animal {
public:
    void speak() const override {
        cout << "Meow!" << endl;
    }
};

int main() {
    // Dynamic dispatch using pointers to base class
    vector<Animal*> zoo;
    zoo.push_back(new Dog());
    zoo.push_back(new Cat());

    for (const auto& animal : zoo) {
        animal->speak(); // Correct version called at runtime
    }

    // Cleanup
    for (auto animal : zoo) {
        delete animal;
    }
    
    return 0;
}

/*
 * OOP Nuances:
 * 1. Virtual destructor: Ensures proper cleanup of derived objects via base pointer.
 * 2. 'virtual' keyword: Instructs compiler to use dynamic dispatch (vtable).
 * 3. 'override': Strongly recommended to catch errors in signature mismatch.
 * 4. 'public' inheritance: Defines an "Is-A" relationship.
 * 5. Polymorphism enables flexible, extensible code architectures.
 */
