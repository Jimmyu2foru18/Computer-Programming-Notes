Linux II: Page 9 - Security & Firewall

Proactive Defense

Security in Linux is a proactive, multi-layered effort. You never rely on a single solution; instead, you build a "fortress" around your system. This involves controlling network traffic at the edge (Firewall), strictly limiting who can gain access and how they gain it (SSH Hardening), and constantly monitoring the system for signs of unauthorized activity (Auditing). These measures drastically reduce your attack surface and help detect potential breaches before they cause significant damage.

1. Controlling Network Traffic (UFW)

The Uncomplicated Firewall (UFW) is the easiest way to manage complex firewall rules. Its policy is simple: "Deny all incoming traffic, allow only what's necessary."


# Basic UFW workflow
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh     # Allow incoming SSH (port 22)
sudo ufw enable        # Turn it on

2. Hardening SSH

SSH is the most common target for attackers. You must disable password-based login and switch to SSH Key-based Authentication.

  • Edit /etc/ssh/sshd_config.
  • Set PermitRootLogin no (Never allow root login directly).
  • Set PasswordAuthentication no (Forbid passwords entirely).

Real-World Relationship: The Gated Community

Think of UFW like the Security Gate of a gated community. The gate stays closed by default (deny incoming), and only residents with a specific code (allowed ports) can enter. Think of SSH Hardening like requiring Biometric Scanners instead of simple PIN codes to enter the security office. Even if someone steals a "key," it's useless because it doesn't match the biometric profile (the SSH key). Think of Auditing like Security Cameras—they record everyone who enters and leaves, so if something goes missing, you have a record to investigate later.
