Web Technologies II: Page 1 - PHP: Server-Side Foundations

The Server-Side Brain

While JavaScript makes the browser "alive," PHP (Hypertext Preprocessor) is the powerhouse that runs on the web server. When a user requests a PHP page, the server executes the PHP code before sending the final, plain HTML to the browser. This allows you to generate content dynamically: fetching user data from a database, processing form submissions, or personalizing the page based on who is logged in. PHP is embedded directly into your HTML files.

1. PHP Basics: Syntax & Variables

PHP code must be enclosed in special tags so the server knows which parts to execute.


<?php
  // This is a comment.
  $name = "John Doe"; // Variables start with $
  $age = 30;
  
  echo "Hello, " . $name . "! You are " . $age . " years old.";
?>

Core Data Types:

  • String: Text data (e.g., "Hello").
  • Integer: Whole numbers.
  • Float: Decimal numbers.
  • Boolean: true or false.
  • Array: Ordered list (or key-value map).

2. Control Flow

PHP uses standard control flow structures similar to other C-style languages. This allows you to create dynamic logic, such as showing different HTML content based on whether a user is logged in or not.


if ($isLoggedIn) {
  echo "<h1>Welcome back!</h1>";
} else {
  echo "<button>Login</button>";
}

Real-World Relationship: The Restaurant Kitchen

Think of your Web Server like a Restaurant Kitchen. The Client (the Browser) is the customer in the dining room. The HTML is the menu. When the customer orders a specific meal (requests a PHP page), the waiter takes the order to the kitchen. The chef (PHP) takes raw ingredients (data from the database or variables) and follows a recipe (PHP code) to cook the meal. The chef then sends the finished meal (the rendered HTML) out to the dining room. The customer never sees the kitchen or the chef working—they only see the final, ready-to-eat meal on their table.
