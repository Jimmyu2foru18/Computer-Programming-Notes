DBMS I: Page 4 - SQL Basics: Data Manipulation & Table Joins

Working with Data

In the previous page, we built the structure of our database using DDL. Now, we will learn how to interact with the actual information stored inside those tables. Data Manipulation Language (DML) is the set of SQL commands used to retrieve, add, modify, and delete records. DML is the most common part of SQL that developers use in their daily work, whether they are building a web app, a mobile game, or an analytics dashboard. Furthermore, we will explore Joins, the powerful mechanism that allows us to combine data from multiple tables into a single result set.

1. CRUD Operations: INSERT, SELECT, UPDATE, DELETE

Most data tasks follow the CRUD acronym: Create, Read, Update, and Delete.

INSERT: Adding New Data


INSERT INTO employees (emp_id, name, salary) 
VALUES (101, 'Alice Smith', 75000);

SELECT: Retrieving Data


SELECT name, salary FROM employees 
WHERE salary > 50000 
ORDER BY salary DESC;

UPDATE & DELETE: Modifying and Removing


UPDATE employees SET salary = 80000 WHERE emp_id = 101;
DELETE FROM employees WHERE emp_id = 101;
Warning: Always use a WHERE clause with UPDATE and DELETE! If you forget it, you will update or delete every single row in the table, which is a common disaster for junior DBAs.

2. Connecting the Dots: Table Joins

The true power of a relational database lies in its ability to connect related data. Instead of one giant, messy table, we use multiple clean tables and "join" them during a query.

  • INNER JOIN: Returns only the rows where there is a match in both tables. This is the most common join.
  • LEFT JOIN: Returns all rows from the left table, and matching rows from the right. If there's no match, you get NULLs on the right side.
  • RIGHT JOIN: The opposite of a Left Join.
  • FULL OUTER JOIN: Returns rows when there is a match in either table.

SELECT e.name, d.dept_name
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id;

Real-World Relationship: The Filing System and the Venn Diagram

Think of DML like Working in a Physical Office. SELECT is searching the filing cabinet for a folder. INSERT is adding a new folder to the drawer. UPDATE is pulling out a form and changing a phone number. DELETE is shredding a document. Think of Joins like a Venn Diagram from math class. Inner Join is the middle part where the circles overlap (the people who are both Employees AND assigned to a Department). Left Join is the entire left circle—all employees, including the new ones who don't have a department assigned yet. This logic allows you to pull meaningful stories out of raw, scattered data points.
