C# I: Page 7 - Text Manipulation: Strings & Interpolation

The Power of Text

In almost every application, you will work with text. From user names to log files, Strings are the primary way we represent textual data. In C#, a string is an object, not a primitive, and it comes with a vast array of built-in methods for searching, splitting, and transforming text. One of the most important concepts to understand is that strings are immutable—once created, they cannot be changed. This unique design choice has massive implications for performance and safety in professional software development.

1. String Immutability

When you "change" a string in C#, you aren't actually modifying the original text in memory. Instead, the CLR creates a brand new string object and points your variable to the new location. The old string is left for the Garbage Collector to clean up.


string s = "Hello";
s = s + " World"; // "Hello" is abandoned, a NEW "Hello World" is created
Performance Warning: If you are concatenating strings thousands of times (e.g., in a loop), use StringBuilder. Standard string concatenation in a loop is extremely slow and memory-intensive.

2. String Interpolation & Verbatim Strings

Modern C# provides two specialized ways to define strings that make your code much cleaner.

String Interpolation (The $ operator)

This allows you to embed variables directly into the string using curly braces {}. It is much more readable than old-fashioned concatenation.


string name = "Alice";
int age = 30;
string greeting = $"Hello, {name}! You are {age} years old.";

Verbatim Strings (The @ operator)

This tells C# to ignore escape characters like \n or \t and preserve line breaks exactly as they appear. It is essential for file paths and multi-line text.


string path = @"C:\Users\Documents\data.txt";

3. Essential String Methods

  • Length: Property that returns the number of characters.
  • ToUpper() / ToLower(): Changes the case of the text.
  • Replace(old, new): Swaps all occurrences of a substring.
  • Substring(start, length): Extracts a portion of the string.
  • Contains(): Checks if a substring exists (returns a bool).

Real-World Relationship: The Printing Press and the Fill-in-the-Blank Form

Think of a C# String like a Printed Book. Once the ink is dry, you can't reach in and change one letter. If there's a typo, you have to print a completely new edition of the book (Immutability). This ensures that once someone reads a page, the information won't change while they're looking at it (Safety). Think of String Interpolation like a Mad Libs Form. You have a static template with specific holes ({}). You drop in your nouns and verbs (Variables) to create a custom story on the fly. Finally, think of Verbatim Strings like a "Do Not Disturb" sign: you're telling the C# compiler, "Don't touch my text, don't interpret my backslashes, just leave it exactly as it is."
