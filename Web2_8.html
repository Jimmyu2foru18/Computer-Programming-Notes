Web Technologies II: Page 8 - Professional Deployment & CI/CD

From Code to Production

In the early days, "deployment" meant manually copying files from your laptop to a server using FTP. If you made a mistake, you had to manually fix it on the live server. Today, this is unacceptable. Professional Deployment relies on Automation to ensure that your code is tested, built, and deployed consistently every single time. This is achieved through a CI/CD Pipeline (Continuous Integration / Continuous Deployment). Furthermore, you must manage Environment Variables to ensure that sensitive information—like database passwords—never accidentally ends up in your public code repository.

1. The CI/CD Pipeline

A CI/CD pipeline automates the steps between writing code and deploying it:

  • Continuous Integration (CI): Every time you push code to Git, the pipeline automatically runs tests. If tests fail, the build is stopped.
  • Continuous Deployment (CD): If the tests pass, the code is automatically built and shipped to the live server.

# Example GitHub Actions pipeline snippet
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm install
      - run: npm test

2. Managing Secrets (.env)

Never hardcode API keys or database passwords. Use Environment Variables. Store them in a local .env file (which you add to .gitignore so it's never committed to Git) and access them in your code via `process.env` (Node) or `getenv()` (PHP).

Real-World Relationship: The Automated Manufacturing Line

Think of Manual Deployment like a Craftsman building a car entirely by hand. If the craftsman makes a mistake, the car is flawed. If they are tired, they make mistakes. Think of CI/CD like an Automated Robotic Assembly Line. Every single car is built using the exact same process. If a part doesn't meet quality standards (tests fail), the robot stops the line. The car never leaves the factory until it is perfectly built and tested. Environment Variables are like the Secret Recipes kept in a locked safe—the factory workers know how to *use* the ingredients to make the product, but they don't know the exact chemical formula, ensuring that the company's secret competitive advantage stays safe.
