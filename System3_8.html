System Design III: Page 8 - Security: Auth/AuthZ

Identity and Permissions

In web application security, the most fundamental concepts are Authentication (AuthN) and Authorization (AuthZ). They are often confused but perform entirely different roles. AuthN verifies who a user is (e.g., login credentials), while AuthZ determines what that user is allowed to do (e.g., can they read data, edit profiles, or delete admins?).

1. Protocols & Tools

  • OAuth2/OIDC: The standard for delegated authorization (e.g., "Log in with Google").
  • JWT (JSON Web Token): A secure, stateless way to transmit identity information between parties as a compact, URL-safe JSON object.
