Linux I: Page 6 - File Operations

Managing Your Data

Now that you can navigate the filesystem (using cd, ls, pwd) and understand where things go (FHS), it's time to learn how to actively manage those files. In the Linux terminal, you don't drag-and-drop to copy or move files; you use fast, powerful commands. Be careful: many of these commands, especially when combined with rm, are permanent and cannot be "undone" via a trash bin.

1. The Fundamental Trio

  • cp (Copy): Creates a duplicate of a file or directory. To copy a directory, you must use the -r (recursive) flag.
  • mv (Move/Rename): Used to move a file from one place to another. Since "renaming" is just moving a file to the same location with a new name, mv is also used for renaming files.
  • rm (Remove): Deletes files. Use rm -r to delete a directory and all its contents. Caution: Deleting as root (or via sudo) can instantly destroy your entire system if misused.

# Copying:
cp file.txt backup.txt
cp -r folder/ backup_folder/

# Moving/Renaming:
mv oldname.txt newname.txt
mv file.txt /home/user/documents/

# Removing:
rm file.txt
rm -rf directory_to_delete/ # Forceful recursive removal

2. Safety First

Always use the -i (interactive) flag when learning. It prompts you for confirmation before overwriting or deleting files, which is a lifesaver for beginners.

Real-World Relationship: The Office Administrator

Think of cp like a Photocopier—you make an identical copy of a document. Think of mv like a Filing Clerk—they take a folder from your desk and put it into the cabinet (moving), or they simply change the label on the folder (renaming). Think of rm like a Paper Shredder. When you put a document in the shredder, it's gone forever. The -f flag is like shredding without checking if you really meant to do it, and -r is like shredding an entire filing cabinet at once. Always check the label *before* you feed the paper into the machine.
