C# II: Page 2 - Object-Oriented Programming (OOP): Inheritance & Polymorphism

The Architecture of Reusability

In the physical world, we rarely build everything from scratch. When a company designs a new car, they don't reinvent the "wheel" or the "engine"; they take existing technology and build on top of it. In programming, we do the same thing using Inheritance and Polymorphism. Inheritance allows us to create a new class (the Derived Class) that inherits attributes and methods from an existing class (the Base Class). Polymorphism allows us to treat those different classes as if they were the same type, making our systems incredibly flexible and scalable. Together, these two pillars of OOP are the primary tools for managing large-scale software architecture.

1. Inheritance: The "Is-A" Relationship

In C#, you establish inheritance using the colon (:) symbol. The derived class gets all the public and protected members of the base class. It can then add its own unique members or Override existing ones to change their behavior.

The Code implementation (C#)


class Animal {
    public string Name { get; set; }
    public virtual void MakeSound() {
        Console.WriteLine("Generic animal sound.");
    }
}

class Dog : Animal { // Dog inherits from Animal
    public override void MakeSound() {
        Console.WriteLine("Woof!");
    }
}

The base keyword: Inside a derived class, you can use base.MakeSound() to call the original version of the method from the parent class.

2. Polymorphism: The Art of Many Forms

Polymorphism means "many forms." In C#, it allows you to use a Base Class Reference to point to a Derived Class Object. This is extremely powerful because it allows you to write code that works with any type of Animal, and each animal will automatically perform its own specific behavior.


// Polymorphic usage
Animal myPet = new Dog();
myPet.MakeSound(); // Output: Woof! (Even though the reference is 'Animal')

List<Animal> zoo = new List<Animal> { new Dog(), new Cat() };
foreach (var a in zoo) {
    a.MakeSound(); // Each animal responds in its own way
}

3. Type Checking: is and as

Sometimes when using polymorphism, you need to know exactly which derived type you are working with. C# provides two safe ways to do this:

  • is: Checks if an object is of a certain type (returns bool). if (animal is Dog d) { d.Fetch(); }
  • as: Tries to convert the object to a type. If it fails, it returns null instead of crashing the program.

Real-World Relationship: Evolution and the Universal Remote

Think of Inheritance like Biological Evolution. Every animal inherits a set of basic blueprints from its ancestors (DNA). A wolf inherits the ability to breathe and eat from the category of "Mammals." Over time, the "Dog" class diverged, inheriting the basic structure but "overriding" behaviors to be more friendly to humans. Think of Polymorphism like a Universal TV Remote. The remote has a "Volume Up" button (the Method Call). It doesn't need to know if your TV is a Sony, a Samsung, or an LG. It just sends the "Volume Up" signal, and each device (the Object) responds in its own specific way (the Implementation). This allow you to control 100 different devices using only one set of buttons (the Interface).
