Java I: Page 9 - Introduction to OOP: Classes & Objects

A New Way of Organizing: Objects

So far, we have written code in a "procedural" way—writing individual instructions and methods that work on data. However, as software becomes more complex, it's often more natural to organize our code around Objects. This is known as Object-Oriented Programming (OOP). In Java, everything revolves around objects. We group related data (State) and behaviors (Methods) together into a single unit. This allows us to model real-world things—like a Car, a Bank Account, or a User—directly in our code.

The Blueprint vs. The Reality

To create objects, we first need a Class. A class is a blueprint or a template. It defines what data an object will have and what it can do, but it doesn't contain the data itself. An Object (or Instance) is the actual thing built from that blueprint.

CLASS: Car (Blueprint)
Attributes: brand, color, speed
Methods: accelerate(), brake()
→
OBJECT: myToyota (Instance)
brand: "Toyota"
color: "Red"
speed: 60

Defining a Class and Creating Objects

A class typically contains Instance Variables (to store the object's state) and Methods (to define the object's behavior).


public class Dog {
    // State (Instance Variables)
    String breed;
    int age;

    // Behavior (Method)
    public void bark() {
        System.out.println("Woof!");
    }
}

// Creating and using an object
Dog myDog = new Dog(); // The 'new' keyword creates the instance
myDog.breed = "Golden Retriever";
myDog.age = 3;
myDog.bark(); // Output: Woof!

The this Keyword

Within a class method, the this keyword refers to the current instance of the object. It is most commonly used to distinguish between instance variables and parameters with the same name.

Real-World Relationship: The Cookie Cutter and the Passport

Think of a Class like a Cookie Cutter. The cutter itself defines the shape (the methods and the slots for data) but it isn't something you can eat. Every time you press that cutter into the dough (use the new keyword), you create a Cookie (an Object). You can decorate each cookie differently—one has sprinkles, one has chocolate chips (different instance variable values)—but they all share the same underlying shape. Another example is a Passport Office: the form you fill out is the Class. Once the office prints the actual Passport with your photo and name on it, that physical book is the Object. It has the structure of every other passport, but it belongs uniquely to you.
