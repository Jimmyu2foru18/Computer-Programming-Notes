C# II: Page 9 - Advanced Concepts: Generics & Reflection

The Metadata Layer

In our final pages of the C# Masterclass, we explore the features that allow you to write code that is extremely flexible and dynamic. Generics allow you to write code that works with any data type while maintaining 100% type safety. Reflection allows your code to "look at itself"—it can inspect classes, methods, and properties at runtime, and even create new objects or call methods on the fly. These features are the secret behind every major .NET framework, from Entity Framework to ASP.NET Core.

1. Generics: Code Reusability and Safety

Before generics, if you wanted a "Box" class for an int and a "Box" class for a string, you had to write two separate classes. With Generics, you write it once using a placeholder <T>.


public class DataHolder<T> {
    public T Value { get; set; }
}

// Usage
var intBox = new DataHolder<int> { Value = 10 };
var strBox = new DataHolder<string> { Value = "Hello" };

Constraints: You can restrict T to be a specific type using the where keyword (e.g., where T : class).

2. Reflection: Inspecting Types at Runtime

Reflection is a powerful technique that lets you inspect the metadata of your assembly. You can find out what methods a class has, what its attributes are, and invoke them without knowing them at compile-time.


using System.Reflection;

Type t = typeof(MyClass);
Console.WriteLine($"Class Name: {t.Name}");

foreach (MethodInfo m in t.GetMethods()) {
    Console.WriteLine($"Method: {m.Name}");
}

3. Attributes: Decorating Your Code

Attributes are metadata that you "attach" to classes, methods, or properties. They don't change the logic itself, but they provide instructions to other parts of the system (like telling a JSON serializer to ignore a property). We often use Reflection to read these attributes.


[Serializable]
public class User {
    [Obsolete("Use NewMethod instead")]
    public void OldMethod() { /* ... */ }
}

Real-World Relationship: The Multi-tool and the Detective

Think of Generics like a Universal Screwdriver with a set of 50 different heads. The screwdriver body (the Class Logic) is the same, but you can "Snap-in" any head (the Type T) you need. You don't have to carry 50 different screwdrivers. Think of Reflection like a Detective with a Flashlight. The detective walks into a dark room (the Assembly) and shines the light on objects. They can see the serial numbers (Metadata), see what tools are in the room (Methods), and even use those tools (Invoke) even if they've never been in that room before. Finally, think of Attributes like Post-it Notes on a Blueprint: "Fragile," "Obsolete," or "Internal Use Only." The builder (the Compiler) sees the notes and changes how they handle those specific parts of the building.
