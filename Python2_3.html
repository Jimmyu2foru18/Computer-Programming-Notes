Python II: Page 3 - The Ecosystem: Dependency Management & Environments

Standing on the Shoulders of Giants

One of the reasons Python is so dominant in the industry is its massive ecosystem. You don't have to write your own web server, image processor, or machine learning algorithm from scratch. Thousands of developers have already written these tools and shared them with the world. However, managing these "third-party" libraries can become complex. In this page, we will learn the professional standard for managing dependencies and creating isolated environments for your projects.

pip: The Python Package Installer

pip is the standard tool for installing and managing additional libraries that are not part of the Python standard library. It connects to the Python Package Index (PyPI), which hosts hundreds of thousands of packages.


# Installing a single package
pip install requests

# Installing multiple packages
pip install pandas numpy matplotlib

Once a package is installed, you can use it in your code like any other module: import requests.

Virtual Environments: Project Isolation

Imagine you have two projects. Project A needs version 1.0 of a library, and Project B needs version 2.0. If you install them globally on your computer, they will conflict. The solution is Virtual Environments (venv). A virtual environment is an isolated folder that has its own copy of Python and its own set of installed packages.

Setting Up a Virtual Environment


# 1. Create the environment (run once per project)
python -m venv myenv

# 2. Activate it (Windows)
myenv\Scripts\activate

# 2. Activate it (macOS / Linux)
source myenv/bin/activate

Once activated, any pip install command will only affect that specific project, keeping your global Python installation clean.

Reproducibility: requirements.txt

When you share your project with a teammate or deploy it to a server, you don't send the thousands of files inside your myenv folder. Instead, you send a requirements.txt file—a simple manifest of every package your project needs.


# Generate the requirements file
pip freeze > requirements.txt

# Someone else can then install everything in one go:
pip install -r requirements.txt

Real-World Relationship: The Laboratory and the Manifest

Think of your Global Python Installation like a Community Kitchen. If everyone leaves their half-used ingredients and dirty pots everywhere, it becomes a mess and no one can cook safely. A Virtual Environment is like a Private Laboratory. You lock the door and only bring in the exact chemicals and equipment you need for this experiment. If the experiment fails and there's a spill, you just throw the whole room away and start over without affecting the rest of the building. Finally, think of requirements.txt like a Scientific Manifest. It's a list of every ingredient and tool used in your successful experiment. By giving that list to another scientist in another city, they can perfectly recreate your results in their own private lab.
