Java II: Page 2 - Object-Oriented Programming (OOP): Inheritance Basics

The Power of a Legacy

In the physical world, we rarely build everything from scratch. When a company designs a new smartphone, they don't reinvent the concepts of a "battery" or a "screen"; they take existing technology and build on top of it. In programming, we do the same thing using Inheritance. Inheritance allows us to create a new class (the Subclass or Derived Class) that inherits all the attributes and methods of an existing class (the Superclass or Base Class). This promotes code reuse and establishes a logical hierarchy in your software.

The "Is-A" Relationship

The easiest way to know if you should use inheritance is to ask: "Is the subclass a type of the superclass?"

  • A Dog is an Animal.
  • A Car is a Vehicle.
  • A Manager is an Employee.

If the answer is yes, then inheritance is likely the right choice. In Java, we use the extends keyword to establish this link.

Basic Inheritance Syntax


class Animal {
    String name;
    public void eat() {
        System.out.println("Animal is eating...");
    }
}

// Dog inherits EVERYTHING from Animal
class Dog extends Animal {
    public void bark() {
        System.out.println("Woof!");
    }
}

Dog myDog = new Dog();
myDog.name = "Buddy"; // Attribute from Animal
myDog.eat();         // Method from Animal
myDog.bark();        // Method from Dog

Using super for Parent Logic

A subclass often needs to call its parent's constructor or methods. For this, Java provides the super keyword. This is especially important for initialization.


class Dog extends Animal {
    String breed;

    public Dog(String name, String breed) {
        super(); // Call the parent constructor (optional if default)
        this.name = name;
        this.breed = breed;
    }
}
Expert Note: Java only supports Single Inheritance for classes. A class can only extend ONE superclass. This prevents the "Diamond Problem" of conflicting parents, though a class can implement multiple interfaces (which we will cover later).

Real-World Relationship: Evolution and Smartphone Generations

Think of Inheritance like Biological Evolution. Every animal inherits a set of basic blueprints from its ancestors (DNA). A wolf inherits the ability to breathe, eat, and sleep from the general category of "Mammals." Over thousands of years, the "Dog" class diverged from the "Wolf" class, inheriting the wolf's basic structure but adding new behaviors to be more friendly to humans. Another example is Smartphone Generations: the iPhone 15 inherits the basic concepts of the iPhone 1 (a touch screen, an internet connection, a camera), but it specializes the hardware and overrides the software to be faster. You don't build a new phone by inventing the "Touch Screen" again; you extend the existing smartphone blueprint.
