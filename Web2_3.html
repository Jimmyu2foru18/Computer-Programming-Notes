Web Technologies II: Page 3 - Node.js & Express

JavaScript Everywhere

Traditionally, JavaScript could only run inside a web browser. Node.js changed that by providing a runtime environment that allows JavaScript to run directly on your computer or server. This enables you to use the same language—JavaScript—for both your frontend and your backend. Express.js is the most popular framework built on top of Node.js. It simplifies the process of creating web servers, handling HTTP requests, and building robust REST APIs, making it a favorite for modern full-stack development.

1. Node.js & NPM

Node.js is built on Google's V8 engine, making it exceptionally fast. A huge part of the Node ecosystem is NPM (Node Package Manager). NPM is the world's largest software registry, allowing you to easily install thousands of pre-built "packages" (like Express, database drivers, or authentication tools) to speed up your development.


# Initialize a new project
npm init -y
# Install express
npm install express

2. Express: Making Servers Simple

Without a framework, creating a server in Node requires a lot of code. Express reduces this to just a few lines.


const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello from Express!');
});

app.listen(3000, () => console.log('Server running!'));

Routing:

Express makes defining "routes" (URL paths like /users or /login) intuitive, allowing you to map URLs directly to specific functions that handle the business logic.

Real-World Relationship: The JavaScript Factory

Think of Node.js as a Factory Building that lets you run your "JavaScript Machinery" anywhere, not just in the "Browser Showroom." Think of NPM as the Industrial Supply Catalog. Instead of building every screw, gear, and motor from scratch, you can browse the catalog (NPM) and order pre-manufactured parts (libraries/packages) to assemble your machinery faster. Think of Express as the Assembly Line Supervisor. They organize the factory floor, direct the flow of incoming materials (HTTP Requests), and ensure the right machinery (your route handler functions) is triggered to process those materials and output a finished product (the HTTP Response).
