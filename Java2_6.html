Java II: Page 6 - Project Architecture: Packages & Access Control

Organizing at Scale

In the physical world, a large library doesn't just throw all its books into one giant pile. It organizes them by floor, then by genre, and finally by author. In Java, as your projects grow from a few files to hundreds or thousands of classes, you need a similar system for organization. Packages are Java's namespaces—they provide a way to group related classes and interfaces, prevent naming conflicts, and control visibility between different parts of your application.

1. Defining and Importing Packages

To put a class in a package, you use the package keyword as the very first line of your code. By convention, package names are all lowercase and use a Reversed Domain Name (e.g., com.google.search). This ensures that your package names are globally unique.


// File: src/com/myapp/utils/MathHelper.java
package com.myapp.utils;

public class MathHelper {
    public static int add(int a, int b) { return a + b; }
}

To use this class in another file, you must import it.


import com.myapp.utils.MathHelper; // Specific import
import java.util.*;               // Wildcard import (all classes in package)

2. Access Control Revisited: The Package Level

In Module 1, we touched on access modifiers. Now that we understand packages, we can see how they really work. The Default access modifier (no keyword used) is often called "Package-Private."

  • Package-Private: Visible only to classes inside the same package. This is perfect for "helper" classes that shouldn't be exposed to the rest of the app.
  • Protected: Visible to classes in the same package AND any subclasses in different packages. This is the "bridge" between packages for inheritance.

3. Static Imports

If you find yourself using a lot of static methods from a class (like Math.sqrt), you can use a Static Import to bring those methods directly into your namespace so you don't have to type the class name every time.


import static java.lang.Math.*;

double result = sqrt(pow(3, 2) + pow(4, 2)); // No 'Math.' needed!

Real-World Relationship: The Global Postal System and the Folder System

Think of a Package like the International Postal Address System. If you send a letter to "123 Main St," it will never arrive because there are thousands of Main Streets. But if you address it to USA.NewYork.NYC.MainSt.123 (Reversed Domain Name), it is globally unique. Packages ensure that your class User doesn't collide with the User class in a library you downloaded from Google. Another example is your Computer's Folder System: you don't put every photo, document, and video on your desktop. You use folders (Packages) to keep things organized. "Default Access" is like having a Shared Household Folder: everyone in the house (the Package) can see it, but the neighbors (other Packages) are locked out.
