C# I: Page 5 - Control Flow: Iteration with Loops (For & Foreach)

The Power of Automation

In programming, we rarely perform an action just once. We process lists of 10,000 customers, update every pixel on a screen 60 times per second, or wait for a user to enter the correct password. Doing this manually is impossible. Loops allow us to write a block of code once and tell the computer to repeat it as many times as necessary. C# provides several distinct types of loops, each designed for a specific scenario. Mastering these is the key to writing efficient code that can handle large amounts of data without repeating yourself.

1. The for Loop: Definite Iteration

The traditional for loop is used when you know exactly how many times you want the code to run. It consists of three parts: initialization, condition, and iteration (increment/decrement).


for (int i = 0; i < 5; i++) {
    Console.WriteLine($"Iteration number: {i}");
}

Common Use Case: Stepping through a numeric range or accessing array elements by their index.

2. The foreach Loop: The Collection Standard

In C#, the foreach loop is the preferred way to iterate over a collection (like an array or a list). It is cleaner and safer than a standard for loop because you don't have to manage a counter variable or worry about "Off-by-one" errors.


string[] names = { "Alice", "Bob", "Charlie" };

foreach (string name in names) {
    Console.WriteLine($"Hello, {name}!");
}
Expert Note: In a foreach loop, the iteration variable (name in the example above) is read-only. If you need to modify the items in the collection while looping, you must use a standard for loop.

3. Indefinite Iteration: while and do-while

These loops repeat as long as a condition remains true. Use them when you don't know the end point in advance.

  • while: Checks the condition before running the block. If it's false at the start, the loop never runs.
  • do-while: Runs the block first, then checks the condition. This guarantees the code will run at least once.

int count = 0;
while (count < 3) {
    Console.WriteLine(count);
    count++;
}

4. Loop Control: break and continue

Sometimes you need to change the flow of a loop from the inside.

  • break: Immediately exits the loop entirely.
  • continue: Skips the rest of the current iteration and jumps back to the top to start the next one.

Real-World Relationship: The Assembly Line and the Loading Screen

Think of a For Loop like an Industrial Assembly Line. You have 1,000 cars (the sequence) on a conveyor belt. The robotic arm (your code) does the same job on every car that passes. It knows exactly when the shift is over because it knows how many cars were scheduled. Think of a While Loop like a Loading Screen in a video game. The screen keeps spinning while the game data is still loading. The computer doesn't know if it will take 5 seconds or 50, so it just keeps looping until the "loading finished" flag is True. Finally, think of break like an Emergency Stop Button on that assembly line—if a sensor detects a fire (a specific condition), the whole machine stops immediately, no matter how many cars are left.
