Linux II: Page 4 - Shell Scripting Basics

Automating the Routine

If you find yourself typing the same series of commands every day, you are wasting time that could be better spent on more interesting problems. Shell Scripting allows you to bundle those commands into a single file and execute them all at once. A shell script is simply a text file containing a list of CLI commands, plus some basic programming logic like variables, loops, and conditional statements. It is the fundamental building block for server automation and administration.

1. The Script Skeleton

Every script starts with a Shebang—a special line that tells the system which interpreter to use (usually Bash). You also need to make the script executable using chmod +x.


#!/bin/bash
# A simple greeting script

NAME="Admin"
echo "Hello, $NAME!"
        

# Make it executable
chmod +x script.sh
# Run it
./script.sh

2. Adding Logic

Scripts are more than just lists of commands; they can make decisions and repeat tasks:

  • Conditionals (if/case): Execute code only if specific criteria are met (e.g., check if a file exists before trying to delete it).
  • Loops (for/while): Perform the same task on multiple files or directories automatically.

Real-World Relationship: The Recipe Book

Think of Shell Scripting like writing a Recipe Book for a robot assistant. Each command you type in the terminal is one step (e.g., "chop onion"). A script is the full recipe (e.g., "make soup") that chains those steps together in a specific order. Variables are like the ingredients listed on the side, Conditionals are like checking "if the pot is hot, add water," and Loops are like "stir the soup 50 times." By writing down the recipe once, your robot assistant can make the soup perfectly every single time without you needing to stand there and supervise the process.
