System Design III: Page 2 - Design: Security

Security by Design

Security cannot be bolted on at the end of a project. Secure System Design is the process of embedding security into the architectural foundation. This involves Threat Modeling—proactively identifying potential attackers and vulnerabilities before a single line of code is written—and implementing Defense-in-Depth, where multiple independent security layers protect the system so that the failure of one does not lead to total compromise.

1. Key Architectural Principles

  • Least Privilege: Every component (or user) has only the absolute minimum permissions required to perform its task.
  • Secure-by-Default: The system starts in its most secure state, requiring explicit actions to enable less secure features.
  • Defense-in-Depth: If the firewall fails, the application authentication catches the threat. If that fails, the database permissions catch it.
