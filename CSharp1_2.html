C# I: Page 2 - Variables, Data Types & Primitives

Storing Data in C#

In any program, you need a way to store and retrieve information. In C#, we use Variables for this. Because C# is a Strongly Typed language, every variable must have a declared "Data Type." This type tells the computer how much memory to set aside and what kind of operations are allowed (e.g., you can't subtract a word from a number). C# provides a rich set of built-in types, ranging from simple numbers to complex text strings. Understanding these is the first step toward writing efficient and safe code.

1. Simple Types: The Primitives

Primitive types are the most basic data units. They are value types, meaning they store the actual data directly in memory (usually on the Stack).

Type Description Size Example
int Whole numbers 32-bit int age = 30;
double High-precision decimals 64-bit double pi = 3.14159;
decimal Financial/Money accuracy 128-bit decimal price = 19.99m;
bool Logic (True/False) 8-bit bool isAlive = true;
char Single character 16-bit char grade = 'A';
Pro Tip: When working with money, always use the decimal type with the m suffix. Standard floats and doubles can have tiny rounding errors that lead to missing cents in financial calculations.

2. Modern Convenience: var and Type Inference

Starting with C# 3.0, you can use the var keyword to let the compiler "infer" the type based on the value you provide. This makes your code cleaner, especially when dealing with complex object names, while still maintaining 100% type safety.


var name = "Alice";      // Inferred as string
var quantity = 10;       // Inferred as int
var ratio = 0.5;         // Inferred as double

3. Constants and Readonly

If you have a value that should never change (like PI or a Company_Name), you use the const keyword. For values that are set once when the program starts but then stay the same, you use readonly.


const double PI = 3.14159;
// PI = 3.15; // ERROR: Cannot change a constant

Real-World Relationship: The Toolbox and the Storage Bins

Think of C# Variables like a Professional Storage System. In a messy garage (Dynamic languages), you might throw everything in one big box. In a C# garage, you have specific bins for specific things. Integers are small bins for counting nails. Doubles are graduated cylinders for measuring liquids accurately. Strings are long tubes for storing scrolls of text. The var keyword is like an Automatic Label Maker: you drop an item into a bin, and the machine instantly sees what it is and slaps a permanent label on it. This organization ensures that you never accidentally try to store motor oil (a number) in a cardboard box (a string), preventing a disaster before it ever happens.
