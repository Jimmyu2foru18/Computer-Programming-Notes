System Design II: Page 6 - DB: Indexing & Tuning

Optimizing for Speed

A database can contain millions of records. Without Indexing, the database must perform a "full table scan" to find a specific row—reading every single entry from disk, which is incredibly slow. An Index is a data structure (like a B-Tree or Hash Table) that stores a small, sorted portion of the data, allowing the database to jump directly to the target record. Performance Tuning is the ongoing process of analyzing queries and creating the right indexes to ensure the system remains responsive as it grows.

1. The Power of Indexes

  • B-Trees: The standard index structure for SQL databases; allows for efficient range searches.
  • Composite Indexes: Indexes on multiple columns (e.g., (lastname, firstname)). Order matters!
  • Full-Text Indexes: Optimized for searching long text fields (e.g., article content).

-- Example: Creating an index
CREATE INDEX idx_user_email ON users(email);
