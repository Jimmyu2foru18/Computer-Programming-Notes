## Introduction

Data structures and algorithms form the foundation of computer science and software engineering. They are essential tools that enable efficient problem-solving and program optimization. This comprehensive guide is designed for undergraduate students to understand these concepts thoroughly.

**Data structures** are specialized formats for organizing, storing, and manipulating data. They provide efficient ways to access and modify information, which is crucial for developing optimized software solutions.

**Algorithms** are step-by-step procedures or formulas for solving problems. They define the logical sequence of operations needed to complete a specific task.

The choice of data structure and algorithm significantly impacts a program's performance and efficiency. This guide covers fundamental concepts, common data structures, popular algorithms, and practical implementation examples to provide a strong foundation for efficient problem-solving and coding.

## Basics

### Data Structures Overview

Data structures are ways to organize data for efficient access and modification. When selecting a data structure, consider:

- **Time Complexity**: How long operations take as data size increases
- **Space Complexity**: How much memory is used as data size increases
- **Functionality**: What operations need to be supported

#### Types of Data Structures:

1. **Primitive Data Structures**: 
   - Basic building blocks directly supported by programming languages
   - Examples: integers, floats, characters, booleans

2. **Abstract Data Types (ADTs)**:
   - Theoretical constructs that define operations without specifying implementation
   - Examples: lists, stacks, queues, maps

### Algorithms Overview

Algorithms are sets of instructions to accomplish specific tasks. They can be analyzed based on:

- **Correctness**: Does the algorithm solve the problem correctly?
- **Efficiency**: How well does it use computational resources?
- **Simplicity**: Is it easy to understand and implement?
- **Optimality**: Is it the best possible solution?

### Asymptotic Analysis (Big O Notation)

Big O notation provides an upper bound on the growth rate of an algorithm's time or space complexity as the input size increases.

| Notation | Name | Description | Example Algorithms |
|----------|------|-------------|-------------------|
| O(1) | Constant | Execution time doesn't depend on input size | Array access, Hash table insertion |
| O(log n) | Logarithmic | Execution time increases logarithmically | Binary search |
| O(n) | Linear | Execution time increases linearly | Linear search |
| O(n log n) | Log-linear | Common in efficient sorting algorithms | Merge sort, Quicksort |
| O(n²) | Quadratic | Often seen in simple sorting algorithms | Bubble sort, Insertion sort |
| O(2ⁿ) | Exponential | Execution time doubles with each addition to input | Recursive Fibonacci |
| O(n!) | Factorial | Very inefficient | Brute force traveling salesman |

Understanding these complexities helps in selecting the most efficient algorithm for a given problem, especially as data sizes grow.

## Searching Algorithms

### Linear Search

Linear search is the simplest searching algorithm that sequentially checks each element in a list until the target is found or the end is reached.

**Algorithm:**
1. Start at the beginning of the list
2. Compare each element to the target
3. If found, return the index
4. If not found after checking all elements, return -1

**Time Complexity:**
- Worst case: O(n) - target is the last element or not present
- Best case: O(1) - target is the first element

**Python Implementation:**
```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i  # Return the index if found
    return -1  # Return -1 if not found
```

**When to use:**
- Unsorted data
- Small data sets
- When simplicity is more important than efficiency

### Binary Search

Binary search is an efficient algorithm for finding an item in a sorted list by repeatedly dividing the search interval in half.

**Algorithm:**
1. Start with the entire sorted list
2. Find the middle element
3. If the target equals the middle element, return the index
4. If the target is less than the middle element, search the left half
5. If the target is greater than the middle element, search the right half
6. Repeat until the target is found or the search interval is empty

**Time Complexity:**
- Worst/Average case: O(log n)
- Best case: O(1) - target is the middle element

**Python Implementation:**
```python
def binary_search(arr, target):
    low = 0
    high = len(arr) - 1
    while low <= high:
        mid = (low + high) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1
    return -1  # Return -1 if not found
```

**When to use:**
- Sorted data
- Large data sets
- When efficiency is important

## Sorting Algorithms

### Bubble Sort

Bubble sort repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order.

**Algorithm:**
1. Iterate through the list multiple times
2. In each iteration, compare adjacent elements and swap if needed
3. After each iteration, the largest element "bubbles" to the end
4. Repeat until no more swaps are needed

**Time Complexity:**
- Worst/Average case: O(n²)
- Best case: O(n) - already sorted

**Python Implementation:**
```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        # Flag to optimize if array is already sorted
        swapped = False
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        # If no swapping occurred in this pass, array is sorted
        if not swapped:
            break
    return arr
```

**When to use:**
- Educational purposes
- Small data sets
- Nearly sorted data

### Insertion Sort

Insertion sort builds the final sorted array one item at a time by comparing each element with the already sorted portion and inserting it at the correct position.

**Algorithm:**
1. Start with the second element
2. Compare it with elements to its left
3. Insert it at the correct position in the sorted portion
4. Repeat for all elements

**Time Complexity:**
- Worst/Average case: O(n²)
- Best case: O(n) - already sorted

**Python Implementation:**
```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        # Move elements greater than key one position ahead
        while j >= 0 and key < arr[j]:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

**When to use:**
- Small data sets
- Nearly sorted data
- Online algorithms (where data arrives one piece at a time)

### Merge Sort

Merge sort is a divide-and-conquer algorithm that divides the list into smaller sublists, recursively sorts them, and then merges them back together.

**Algorithm:**
1. Divide the list into two halves
2. Recursively sort each half
3. Merge the sorted halves into a single sorted list

**Time Complexity:**
- All cases: O(n log n)

**Python Implementation:**
```python
def merge_sort(arr):
    if len(arr) > 1:
        mid = len(arr) // 2
        left = arr[:mid]
        right = arr[mid:]
        
        # Recursive calls
        merge_sort(left)
        merge_sort(right)
        
        # Merge
        i = j = k = 0
        while i < len(left) and j < len(right):
            if left[i] < right[j]:
                arr[k] = left[i]
                i += 1
            else:
                arr[k] = right[j]
                j += 1
            k += 1
            
        # Check for remaining elements
        while i < len(left):
            arr[k] = left[i]
            i += 1
            k += 1
            
        while j < len(right):
            arr[k] = right[j]
            j += 1
            k += 1
    return arr
```

**When to use:**
- Large data sets
- When stable sorting is required
- When worst-case performance matters

### Quick Sort

Quicksort is a divide-and-conquer algorithm that picks a pivot element and partitions the array around it.

**Algorithm:**
1. Choose a pivot element
2. Partition the array into elements less than the pivot and elements greater than the pivot
3. Recursively sort the subarrays

**Time Complexity:**
- Average case: O(n log n)
- Worst case: O(n²) - poor pivot selection
- Best case: O(n log n)

**Python Implementation:**
```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + middle + quick_sort(right)
```

**When to use:**
- Large data sets
- When average-case performance matters more than worst-case
- In-place sorting is needed (with a different implementation)

## Linear Data Structures

### Array Lists (Dynamic Arrays)

Array lists (or dynamic arrays) are resizable arrays that automatically grow when elements are added beyond capacity.

**Key Features:**
- Random access (O(1))
- Contiguous memory allocation
- Automatic resizing

**Operations:**
- `append(element)`: Add element to end - Amortized O(1)
- `insert(index, element)`: Insert at specific index - O(n)
- `remove(element)`: Remove first occurrence - O(n)
- `get(index)`: Retrieve element at index - O(1)
- `set(index, element)`: Update element at index - O(1)

**When to use:**
- Random access is frequent
- Size changes are infrequent
- Memory locality is important

### Linked Lists

Linked lists store elements in nodes, where each node contains data and a reference to the next node.

**Types:**
- **Singly Linked List**: Each node points to the next node
- **Doubly Linked List**: Each node points to both next and previous nodes
- **Circular Linked List**: Last node points back to the first node

**Operations:**
- `insert_at_beginning(element)`: O(1)
- `insert_at_end(element)`: O(n) or O(1) with tail pointer
- `remove(element)`: O(n)
- `search(element)`: O(n)

**Python Implementation:**
```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
        
    def insert_at_beginning(self, data):
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node
        
    def print_list(self):
        current = self.head
        while current:
            print(current.data, end=" ")
            current = current.next
        print()
```

**When to use:**
- Frequent insertions/deletions
- No need for random access
- Memory allocation is a concern

### Stacks

A stack is a linear data structure that follows the Last-In-First-Out (LIFO) principle.

**Operations:**
- `push(element)`: Add to top - O(1)
- `pop()`: Remove from top - O(1)
- `peek()`: View top element without removing - O(1)
- `isEmpty()`: Check if stack is empty - O(1)

**Python Implementation:**
```python
class Stack:
    def __init__(self):
        self.items = []
        
    def push(self, item):
        self.items.append(item)
        
    def pop(self):
        if not self.isEmpty():
            return self.items.pop()
        return None
        
    def peek(self):
        if not self.isEmpty():
            return self.items[-1]
        return None
        
    def isEmpty(self):
        return len(self.items) == 0
```

**Applications:**
- Function call management (call stack)
- Expression evaluation
- Backtracking algorithms
- Undo functionality

### Queues

A queue is a linear data structure that follows the First-In-First-Out (FIFO) principle.

**Operations:**
- `enqueue(element)`: Add to rear - O(1)
- `dequeue()`: Remove from front - O(1)
- `peek()`: View front element without removing - O(1)
- `isEmpty()`: Check if queue is empty - O(1)

**Python Implementation:**
```python
class Queue:
    def __init__(self):
        self.items = []
        
    def enqueue(self, item):
        self.items.insert(0, item)
        
    def dequeue(self):
        if not self.isEmpty():
            return self.items.pop()
        return None
        
    def peek(self):
        if not self.isEmpty():
            return self.items[-1]
        return None
        
    def isEmpty(self):
        return len(self.items) == 0
```

**Applications:**
- Task scheduling
- Breadth-first search
- Print queue management
- Message queues in distributed systems

## Hash Tables

Hash tables store key-value pairs and use a hash function to compute an index for efficient retrieval.

**Key Components:**
- **Hash Function**: Maps keys to indices
- **Collision Handling**: Resolves when different keys map to the same index

**Collision Resolution Techniques:**
1. **Separate Chaining**: Store multiple entries at same index using linked lists
2. **Open Addressing**: Find another slot when collision occurs
   - Linear Probing: Check next slot sequentially
   - Quadratic Probing: Check slots at quadratic distances
   - Double Hashing: Use second hash function to determine step size

**Operations:**
- `insert(key, value)`: Average O(1), Worst O(n)
- `get(key)`: Average O(1), Worst O(n)
- `remove(key)`: Average O(1), Worst O(n)

**Python Implementation:**
```python
class HashTable:
    def __init__(self, size):
        self.size = size
        self.table = [None] * size
        
    def _hash(self, key):
        return hash(key) % self.size
        
    def insert(self, key, value):
        index = self._hash(key)
        if self.table[index] is None:
            self.table[index] = [(key, value)]
        else:
            # Handle collision with separate chaining
            for i, (k, v) in enumerate(self.table[index]):
                if k == key:
                    # Update existing key
                    self.table[index][i] = (key, value)
                    return
            # Key not found, append new key-value pair
            self.table[index].append((key, value))
            
    def get(self, key):
        index = self._hash(key)
        if self.table[index] is not None:
            for k, v in self.table[index]:
                if k == key:
                    return v
        return None
        
    def remove(self, key):
        index = self._hash(key)
        if self.table[index] is not None:
            for i, (k, v) in enumerate(self.table[index]):
                if k == key:
                    del self.table[index][i]
                    return True
        return False
```

**Applications:**
- Database indexing
- Caching
- Symbol tables in compilers
- Associative arrays

## Tree Data Structures

Trees are hierarchical data structures with a root node and child nodes connected by edges.

**Terminology:**
- **Root**: Top-most node
- **Parent**: Node with children
- **Child**: Node connected to a parent
- **Leaf**: Node with no children
- **Subtree**: Tree formed by a node and its descendants
- **Depth**: Number of edges from root to node
- **Height**: Number of edges from deepest leaf to node

### Binary Search Trees

A binary search tree (BST) is a binary tree where each node's left subtree contains values less than the node, and the right subtree contains values greater than or equal to the node.

**Operations:**
- `insert(value)`: Average O(log n), Worst O(n)
- `search(value)`: Average O(log n), Worst O(n)
- `delete(value)`: Average O(log n), Worst O(n)

**Tree Traversal:**
- **Inorder**: Left, Root, Right (gives sorted order in BST)
- **Preorder**: Root, Left, Right
- **Postorder**: Left, Right, Root
- **Level Order**: Visit nodes level by level

**Python Implementation:**
```python
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

class BinarySearchTree:
    def __init__(self):
        self.root = None
        
    def insert(self, data):
        new_node = Node(data)
        if self.root is None:
            self.root = new_node
            return
            
        current = self.root
        while True:
            if data < current.data:
                if current.left is None:
                    current.left = new_node
                    return
                current = current.left
            else:
                if current.right is None:
                    current.right = new_node
                    return
                current = current.right
                
    def search(self, data):
        current = self.root
        while current:
            if data == current.data:
                return True
            elif data < current.data:
                current = current.left
            else:
                current = current.right
        return False
        
    def inorder_traversal(self, node):
        if node:
            self.inorder_traversal(node.left)
            print(node.data, end=" ")
            self.inorder_traversal(node.right)
```

**Applications:**
- Database indexing
- Priority queues
- Expression trees
- Hierarchical data representation

### Balanced Trees

Balanced trees maintain a logarithmic height to ensure efficient operations.

#### AVL Trees

AVL trees are self-balancing BSTs where the height difference between left and right subtrees of any node is at most 1.

**Balance Factor = Height(Left Subtree) - Height(Right Subtree)**

**Rotations:**
- **Left Rotation**: When right subtree is heavier
- **Right Rotation**: When left subtree is heavier
- **Left-Right Rotation**: Left rotation on left child, then right rotation on node
- **Right-Left Rotation**: Right rotation on right child, then left rotation on node

**Time Complexity:**
- All operations: O(log n)

#### Red-Black Trees

Red-Black trees are self-balancing BSTs where each node is colored red or black according to specific rules.

**Properties:**
1. Every node is either red or black
2. The root is black
3. All leaves (NIL) are black
4. If a node is red, both its children are black
5. Every path from a node to its descendant leaves contains the same number of black nodes

**Time Complexity:**
- All operations: O(log n)

**Applications of Balanced Trees:**
- Database systems
- File systems
- Implementing sets and maps in programming languages

### Heaps

A heap is a specialized tree-based data structure that satisfies the heap property.

**Types:**
- **Min-Heap**: Parent node's value ≤ children's values
- **Max-Heap**: Parent node's value ≥ children's values

**Operations:**
- `insert(value)`: O(log n)
- `extract_min()/extract_max()`: O(log n)
- `peek_min()/peek_max()`: O(1)
- `heapify()`: O(n)

**Python Implementation (using heapq):**
```python
import heapq

# Min-Heap example
heap = []
heapq.heappush(heap, 5)
heapq.heappush(heap, 1)
heapq.heappush(heap, 3)
print(heapq.heappop(heap))  # Output: 1 (smallest element)
```

**Applications:**
- Priority queues
- Heap sort
- Graph algorithms (Dijkstra's, Prim's)
- Event-driven simulation

## Graph Data Structures

Graphs consist of vertices (nodes) connected by edges, representing relationships between objects.

**Types:**
- **Directed**: Edges have direction
- **Undirected**: Edges have no direction
- **Weighted**: Edges have associated weights/costs
- **Unweighted**: Edges have no weights

### Graph Representations

1. **Adjacency Matrix**:
   - 2D array where matrix[i][j] indicates edge between vertices i and j
   - Space: O(V²)
   - Edge lookup: O(1)
   - Finding all edges: O(V²)

2. **Adjacency List**:
   - Array/dictionary of lists where each vertex stores its adjacent vertices
   - Space: O(V+E)
   - Edge lookup: O(degree(V))
   - Finding all edges: O(V+E)

### Graph Traversal

1. **Depth-First Search (DFS)**:
   - Explores as far as possible along each branch before backtracking
   - Uses stack (or recursion)
   - Applications: Topological sorting, cycle detection, path finding

2. **Breadth-First Search (BFS)**:
   - Explores all neighbors before moving to next level
   - Uses queue
   - Applications: Shortest path (unweighted), connected components

**Python Implementation (BFS):**
```python
def bfs(graph, start):
    visited = []
    queue = [start]
    
    while queue:
        node = queue.pop(0)
        if node not in visited:
            visited.append(node)
            neighbors = graph[node]
            for neighbor in neighbors:
                queue.append(neighbor)
    
    return visited
```

### Common Graph Algorithms

1. **Shortest Path Algorithms**:
   - **Dijkstra's Algorithm**: For graphs with non-negative edge weights
   - **Bellman-Ford Algorithm**: Handles negative edge weights

2. **Minimum Spanning Tree (MST) Algorithms**:
   - **Kruskal's Algorithm**: Sorts edges by weight and adds if no cycle forms
   - **Prim's Algorithm**: Grows MST from a starting vertex

**Applications:**
- Social networks
- Road networks and navigation
- Web page linking
- Dependency resolution

## Algorithm Design Techniques

### Dynamic Programming

Dynamic programming solves complex problems by breaking them into simpler overlapping subproblems and storing solutions to avoid redundant calculations.

**Key Concepts:**
1. **Overlapping Subproblems**: Subproblems recur multiple times
2. **Optimal Substructure**: Optimal solution built from optimal solutions to subproblems

**Approaches:**
1. **Top-Down (Memoization)**:
   - Start with original problem
   - Recursively solve subproblems
   - Store solutions in memo table

2. **Bottom-Up (Tabulation)**:
   - Start with smallest subproblems
   - Build up to larger problems
   - Store results in a table

**Example: Fibonacci Sequence**

**Naive Recursive (Inefficient):**
```python
def fibonacci_recursive(n):
    if n <= 1:
        return n
    return fibonacci_recursive(n-1) + fibonacci_recursive(n-2)
```

**Dynamic Programming (Memoization):**
```python
def fibonacci_memoization(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fibonacci_memoization(n-1, memo) + fibonacci_memoization(n-2, memo)
    return memo[n]
```

**Dynamic Programming (Tabulation):**
```python
def fibonacci_tabulation(n):
    table = [0] * (n+1)
    table[1] = 1
    for i in range(2, n+1):
        table[i] = table[i-1] + table[i-2]
    return table[n]
```

**Applications:**
- Optimization problems
- Sequence alignment
- Resource allocation
- Shortest path problems
