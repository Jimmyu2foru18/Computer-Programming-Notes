Java II: Page 1 - Object-Oriented Programming (OOP): Encapsulation

The Principle of Data Hiding

In our first module, we learned how to build objects with data and behavior. Now, we must learn the most important rule of professional software design: Encapsulation. Encapsulation is the practice of bundling data (attributes) and methods that operate on that data into a single unit (a class) and restricting direct access to some of the object's components. In the professional world, we never allow outside code to reach in and change an object's internal state directly. Instead, we provide controlled "gates" for interaction.

The Four Access Modifiers

Java provides four levels of visibility to help you enforce encapsulation. Choosing the correct modifier is essential for security and maintenance.

Modifier Visibility Use Case
private Same class only The Default for Data. Hide all attributes here.
default Same package only Used for classes that work closely together in one folder.
protected Package + Subclasses Used when you want children to access parent data.
public Everywhere The Default for Methods. Your object's "Interface."

Controlled Access: Getters & Setters

If we make our data private, how does the rest of the program use it? We provide public Getter and Setter methods. This allows us to add Validation logic. For example, a setAge() method can check if the number is positive before saving it.


public class Student {
    private String name; // Private data
    private double gpa;

    // Getter
    public String getName() {
        return name;
    }

    // Setter with Validation
    public void setGpa(double gpa) {
        if (gpa >= 0.0 && gpa <= 4.0) {
            this.gpa = gpa;
        } else {
            System.out.println("Invalid GPA!");
        }
    }
}

Real-World Relationship: The ATM and the Human Body

Think of Encapsulation like an ATM. The bank doesn't let you walk into the vault and move money around by hand (Direct access to data). Instead, they provide a screen and buttons (The Interface). You can only withdraw() or deposit() by following the rules of the machine. The "Internal State" of the bank's ledger is hidden behind the machine's steel walls. Another example is the Human Body: your internal organs (Data) are encapsulated inside your skin. People can interact with you through your voice and hands (Public methods), but they can't directly reach in and change your heart rate or blood sugar. This "Black Box" design ensures that you can function safely without accidental interference from the outside world.
