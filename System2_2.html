System Design II: Page 2 - Backend: REST API Design

Standardizing Communication

REST (Representational State Transfer) is the standard architectural style for building web APIs. It leverages HTTP's built-in verbs (GET, POST, PUT, DELETE) to manage resources. A well-designed REST API is predictable, stateless, and cacheable, making it easy for frontend developers and third-party systems to consume.

1. Key Principles

  • Resources: Use nouns for URLs (e.g., /users, not /getUsers).
  • Verbs: Use HTTP methods to define actions (GET = read, POST = create, PUT = update, DELETE = remove).
  • Status Codes: Return proper codes: 200 OK, 201 Created, 400 Bad Request, 404 Not Found, 500 Server Error.

GET /api/products/123   # Get product
POST /api/products      # Create product
PUT /api/products/123   # Update product
DELETE /api/products/123 # Delete product
