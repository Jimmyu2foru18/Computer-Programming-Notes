C# I: Page 4 - Control Flow: Decision Making with If & Switch

Giving Your Code a Brain

Until now, our code has run in a straight line from top to bottom. However, real-world applications must be able to make decisions. They need to respond differently depending on user input, data values, or the time of day. This is achieved through Control Flow. In C#, decision-making is primarily handled by Conditional Statements. These statements ask a question (a Boolean check) and execute specific blocks of code only if the answer is true. Mastery of these structures is essential for building everything from simple validation to complex game logic.

1. The if, else if, and else Structure

The if statement is the most common way to branch your code. It checks a condition inside parentheses. If that condition is true, the code inside the curly braces {} runs. You can provide alternative paths using else if and a "catch-all" path using else.


int score = 85;

if (score >= 90) {
    Console.WriteLine("Excellent!");
} else if (score >= 70) {
    Console.WriteLine("Good job.");
} else {
    Console.WriteLine("Keep practicing.");
}

The Ternary Operator: For simple assignments based on a condition, you can use the shorthand ? : syntax: string result = (score >= 60) ? "Pass" : "Fail";

2. The switch Statement

When you need to compare a single variable against many discrete values, a switch statement is often cleaner and more readable than a long chain of if-else blocks.

Traditional Switch


int day = 3;
switch (day) {
    case 1: Console.WriteLine("Monday"); break;
    case 2: Console.WriteLine("Tuesday"); break;
    case 3: Console.WriteLine("Wednesday"); break;
    default: Console.WriteLine("Invalid day"); break;
}

3. Modern C#: Switch Expressions

C# 8.0 introduced a powerful new syntax called Switch Expressions. These are more concise and allow the switch to return a value directly. They use the "Arrow" (=>) syntax and the underscore (_) as the default case.


string dayName = day switch {
    1 => "Monday",
    2 => "Tuesday",
    3 => "Wednesday",
    _ => "Unknown"
};

Real-World Relationship: The Railroad Switch and the Vending Machine

Think of If-Else logic like a Railroad Switch. As the train (the Code Execution) approaches the fork, a sensor checks a condition (e.g., "Is the weight > 10 tons?"). If true, the tracks physically move to the left. If false, they stay to the right. The train can only go down one path at a time. Think of a Switch Statement like a Vending Machine. You press button 'A1' (the Variable Value). The machine doesn't check every button; it jumps straight to the mechanism for 'A1' and drops your snack. The Modern Switch Expression is like a smart vending machine that doesn't just give you a snack, but also updates your digital receipt and prints your balance (returns a value) in one fluid motion.
