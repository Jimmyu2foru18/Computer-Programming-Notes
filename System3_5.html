System Design III: Page 5 - Hosting: Cloud

The Cloud Paradigm

Cloud Hosting transformed how we deploy applications. Instead of maintaining physical servers in a cold, noisy basement, you rent virtualized infrastructure from cloud providers (AWS, Azure, GCP). This allows for on-demand scalability—you can spin up a thousand servers in seconds and shut them down when they are no longer needed, paying only for what you use.

1. Service Models

  • IaaS (Infrastructure as a Service): You manage the OS and middleware (e.g., EC2). Maximum control, maximum management overhead.
  • PaaS (Platform as a Service): You manage the code, the provider manages the OS and runtime (e.g., Elastic Beanstalk). Best balance of control vs. speed.
  • SaaS (Software as a Service): You use the application (e.g., Gmail, Slack). Zero management overhead.
