Java II: Page 8 - Data Persistence: File I/O & NIO

Beyond Volatile Memory

Until now, all the data we've worked with has been stored in RAM. The moment you close your program, that data is gone. To build useful applications, we must be able to save data permanently to the hard drive and read it back later. Java provides two primary ways to do this: the traditional IO (Input/Output) package, which uses streams, and the modern NIO (New I/O) package, which uses a more efficient, path-based approach.

1. Traditional I/O: Streams and Readers

Traditional Java I/O is based on Streams—a continuous flow of data. For text files, we use Readers and Writers. The BufferedReader is the professional choice because it buffers the data in memory, making it much faster than reading one character at a time from the disk.


// Writing to a file (Traditional)
try (PrintWriter writer = new PrintWriter(new FileWriter("output.txt"))) {
    writer.println("Hello from Java!");
} catch (IOException e) {
    e.printStackTrace();
}

// Reading from a file (Traditional)
try (BufferedReader reader = new BufferedReader(new FileReader("output.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}

2. Modern NIO: The Files Utility

Introduced in Java 7, the java.nio package is generally preferred for modern development. It is more concise and better at handling large files and complex metadata. It revolves around the Path and Files classes.


import java.nio.file.*;

// Writing all at once
Path path = Paths.get("modern.txt");
String content = "This is a modern Java file.";
Files.write(path, content.getBytes());

// Reading all at once (into a List)
List lines = Files.readAllLines(path);

3. Choosing the Right Tool

Use BufferedReader if you are processing a massive file line-by-line and don't want to load the whole thing into RAM at once. Use Files.readAllLines() for smaller configuration or data files where convenience is more important than memory footprint.

Real-World Relationship: The Garden Hose and the Shipping Container

Think of Traditional I/O Streams like a Garden Hose. Data flows character-by-character (or drop-by-drop). You can only see what is coming out of the nozzle at this exact second. This is great for continuous tasks like logging. Think of Modern NIO like a Shipping Container. You don't care about the individual drops; you just want to move the entire container from the ship (the Disk) to the warehouse (the RAM) in one highly efficient trip. Finally, think of try-with-resources like a Public Restroom with a Self-Closing Door. You don't have to remember to "close" the file when you're done; the system is designed to lock the door behind you automatically, ensuring the next person (or program) can use the resource safely.
