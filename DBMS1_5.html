DBMS I: Page 5 - SQL Basics: Aggregates, Grouping & Data Insights

Summarizing Data

In our previous pages, we learned how to retrieve individual records. But in the world of business intelligence and data science, we often need to look at the Big Picture. How many customers do we have? What is our total revenue? Which department has the highest average salary? To answer these questions, we use Aggregate Functions. These functions perform a calculation on a set of values and return a single, summarized result. When combined with the GROUP BY clause, they become a powerful tool for generating insights and reports from raw data.

1. The Core Aggregate Functions

SQL provides five main functions that you will use in almost every analytical query:

  • COUNT(): Returns the total number of rows or non-NULL values.
  • SUM(): Calculates the total of a numeric column.
  • AVG(): Calculates the arithmetic mean.
  • MIN() / MAX(): Returns the smallest and largest values in a set.

SELECT COUNT(*) AS total_employees, 
       AVG(salary) AS average_salary, 
       MAX(salary) AS highest_salary 
FROM employees;

2. Organizing Results: GROUP BY and HAVING

The GROUP BY statement groups rows that have the same values into summary rows. For example, "find the number of employees in each department."

The HAVING Clause

A common mistake is trying to use WHERE to filter aggregated results. Because WHERE filters data before it is grouped, we need the HAVING clause to filter after the groups are created.


SELECT dept_id, COUNT(*) 
FROM employees 
GROUP BY dept_id 
HAVING COUNT(*) > 10; -- Only show departments with more than 10 people

3. Dealing with NULLs in Aggregates

It is important to remember that most aggregate functions (except COUNT(*)) ignore NULL values. If you have a column for "Commission" and half the employees have NULL, AVG(commission) will only calculate the average of the people who actually received a commission, which might skew your results.

Real-World Relationship: The Classroom and the Grocery Store

Think of Aggregate Functions like a Teacher Grading a Class. COUNT is the number of students. AVG is the class average. MAX is the "Valedictorian" score. Think of GROUP BY like a Grocery Store Checkout. The store has 10,000 individual items sold (Raw Data). GROUP BY category allows the manager to see the total sales for "Produce," "Dairy," and "Bakery." HAVING is like the manager saying, "Show me only the categories that sold more than $1,000 today." This high-level view is what allows businesses to make strategic decisions rather than getting lost in the details of every single receipt.
