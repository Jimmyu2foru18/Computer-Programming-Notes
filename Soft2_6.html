Software Engineering II: Page 6 - Network Communication: Socket Programming

The Digital Conversation

In the modern era, software rarely lives in isolation. Your phone talks to a server, your web browser talks to a database, and your smart home devices talk to each other. Socket Programming is the foundational technique that enables this network communication between two different computers. A Socket is one endpoint of a two-way communication link between two programs running on a network. It is the "doorway" through which data enters and leaves a process.

1. Core Network Concepts

Before you can write socket code, you must understand two primary identifiers:

  • IP Address: The unique numerical identifier for a device on the network (e.g., 192.168.1.1). It is like the "Mailing Address" of the building.
  • Port Number: A 16-bit number that identifies a specific process or service on that device (e.g., Port 80 for HTTP). It is like the "Suite Number" inside the building.

TCP vs. UDP

Sockets typically use one of two transport protocols:

  1. TCP (Transmission Control Protocol): Connection-oriented. It is reliable, ordered, and error-checked. It ensures that 100% of your data arrives in the correct order. (Used for Web, Email, and Files).
  2. UDP (User Datagram Protocol): Connectionless. It is faster but "unreliable"—some data might be lost or arrive out of order. (Used for Video Streaming, Gaming, and Voice calls).

2. TCP Socket Workflow (Java)

Communication follows a specific handshake between a Server (which waits for requests) and a Client (which initiates the request).

Server-Side Implementation:


// 1. Create ServerSocket on port 12345
ServerSocket server = new ServerSocket(12345);
// 2. Wait and Accept a connection
Socket client = server.accept();
// 3. Setup I/O streams
BufferedReader in = new BufferedReader(new InputStreamReader(client.getInputStream()));
PrintWriter out = new PrintWriter(client.getOutputStream(), true);

Client-Side Implementation:


// 1. Connect to server at IP and Port
Socket socket = new Socket("localhost", 12345);
// 2. Send data
PrintWriter out = new PrintWriter(socket.getOutputStream(), true);
out.println("Hello Server!");

Real-World Relationship: The Telephone and the Postcard

Think of TCP like a Telephone Call. You dial the number (the IP/Port). Someone picks up and says "Hello" (the Handshake). You talk back and forth, ensuring you heard every word correctly. If the signal drops, you know immediately. It is a persistent, reliable connection. Think of UDP like a Postcard. You write a message, drop it in the mailbox, and hope it gets there. There is no "Handshake." You don't know if the person received it unless they send one back. If the mail truck crashes, the message is lost forever. However, sending a postcard is much faster and cheaper than setting up a permanent phone line, which is why we use it for "live" data like a Video Stream where missing one tiny pixel doesn't ruin the whole experience.
