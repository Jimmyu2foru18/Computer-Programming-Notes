Linux II: Page 7 - Logs & Maintenance

The Health of the System

A Linux system is constantly generating data about its health: errors, warnings, login attempts, and application events. These logs are stored in /var/log. If you don't manage these logs, they will eventually consume all your disk space. System Maintenance involves setting up automatic log rotation to keep your disk usage under control, as well as running periodic maintenance tasks to ensure the system remains updated, secure, and performant. Automation is the only way to do this consistently.

1. Log Management

In modern Linux, journalctl is your gateway to system logs. It provides a centralized, queryable interface for all system and application events.

  • journalctl -xe: View the most recent logs with extra context.
  • journalctl -u nginx: View logs for a specific service.
  • journalctl -f: "Follow" the logs in real-time.

To prevent logs from filling your disk, use logrotate. It automatically rotates (moves/compresses/deletes) old logs based on rules defined in /etc/logrotate.conf.

2. Automated Maintenance

Don't manually update and clean your server. Write a script and let cron handle it.


# A simple maintenance script
#!/bin/bash
apt update && apt upgrade -y
apt autoremove -y
find /tmp -type f -atime +7 -delete

Real-World Relationship: The Janitor and the Recycling Bin

Think of Logs like the daily trash generated by a busy office. If you never take the trash out (Log Rotation), eventually the office becomes unusable. Logrotate is the Recycling/Trash collection service that comes by regularly to take the old trash away so the office stays clean. An Automated Maintenance Script is like a Night Cleaning Crew. They come in every night to clean the floors, empty the bins, and prepare the office for a fresh start the next morning, ensuring the workplace is safe and productive without the employees needing to worry about it.
