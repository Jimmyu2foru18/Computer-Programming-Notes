System Design III: Page 4 - Deployment: IaC

Infrastructure as Code

In the past, setting up infrastructure meant manually clicking through cloud consoles or running shell scripts on servers. This is error-prone, hard to audit, and impossible to reproduce reliably. Infrastructure as Code (IaC) is the practice of defining your entire infrastructure (servers, databases, networks, load balancers) in version-controlled configuration files. Tools like Terraform or CloudFormation read these files and automatically provision the exact required infrastructure, ensuring that your production environment is identical to your staging environment.

1. Why IaC?

  • Consistency: Eliminates "configuration drift" where staging and production servers slowly diverge.
  • Reproducibility: Need a new staging environment? Run the script again.
  • Version Control: Every change to your infrastructure is tracked in Git, providing a clear audit trail.
