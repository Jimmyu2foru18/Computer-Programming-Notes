Python II: Page 4 - Object-Oriented Programming (OOP): Classes & Instances

The Blueprint for Reality

In the first half of this course, we focused on "procedural" programming—writing functions that perform tasks. However, as software becomes more complex, it is often better to organize our code around Objects. Object-Oriented Programming (OOP) allows us to group data and the functions that work on that data into a single, cohesive unit. This approach makes code more intuitive, modular, and easier to reuse across different parts of a project.

Defining a Class: The Blueprint

A Class is a template or blueprint. It defines what attributes (data) and methods (behavior) an object will have. For example, a Dog class defines that every dog has a name, a breed, and the ability to bark.

The __init__ Method and self

The __init__ method is the Constructor. it runs automatically when you create a new object. The self parameter represents the specific instance of the class and allows you to assign data to that unique object.


class Dog:
    # Class Variable (Shared by all dogs)
    species = "Canis lupus"

    def __init__(self, name, breed, age):
        # Instance Variables (Unique to each dog)
        self.name = name
        self.breed = breed
        self.age = age
        self.tricks = []

    def bark(self):
        return f"{self.name} says: Woof!"

    def learn_trick(self, trick):
        self.tricks.append(trick)
        return f"{self.name} learned {trick}!"

Class Variables vs. Instance Variables

Understanding the difference between these two is vital for managing state in your programs.

  • Instance Variables: Data that is unique to each object (e.g., each dog has a different name). These are defined inside __init__ using self.
  • Class Variables: Data that is shared by all instances of a class (e.g., all dogs belong to the same species). These are defined directly inside the class but outside any methods.

Custom Representation: The __str__ Method

By default, printing an object looks like <__main__.Dog object at 0x...>. You can make your objects "printable" and human-readable by defining the __str__ dunder method.


    def __str__(self):
        return f"{self.name} is a {self.age}-year-old {self.breed}"

my_dog = Dog("Buddy", "Golden Retriever", 3)
print(my_dog)  # Output: Buddy is a 3-year-old Golden Retriever

Real-World Relationship: The Cookie Cutter and the Passport

Think of a Class like a Cookie Cutter. The cutter itself defines the shape (Attributes and Methods) but isn't a cookie. Every Instance you bake from that cutter is an Object—they all have the same shape, but you can decorate each one differently (Instance Variables). Think of the Constructor (__init__) as the Passport Office. When a new person (Object) is born into the system, the office gives them a name, birthdate, and nationality (Attributes). Finally, think of self like a person's Identity: if I tell 100 people "Stand up," they all look at themselves (self) to see if they are the one being spoken to and then perform the action (the Method).
