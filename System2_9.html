System Design II: Page 9 - Backend: Concurrency

Handling Multiple Requests

A server needs to handle thousands of users at once. If your code is "blocking"—meaning it stops everything to wait for a database query or a slow API—your server will quickly become unresponsive. Concurrency is the ability of your backend to manage multiple tasks simultaneously. Different languages use different models: Multi-threading (Java), Multi-processing (Python/Gunicorn), or an asynchronous Event Loop (Node.js/FastAPI).

1. Concurrency Paradigms

  • Multi-threading: Spawns lightweight threads for each request. Great for CPU-heavy tasks, but complex to manage shared state safely (thread safety).
  • Asynchronous Event Loop: A single-threaded loop that manages I/O asynchronously. When one task waits for a database, the loop instantly switches to handle another user's request. Perfect for I/O-heavy applications.

// Node.js Event Loop Concept
// This doesn't block! It handles other requests while waiting.
const data = await db.query("SELECT * FROM large_table");
