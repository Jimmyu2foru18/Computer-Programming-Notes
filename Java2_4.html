Java II: Page 4 - Object-Oriented Programming (OOP): Interfaces & Contracts

The Pure Interface

In our previous pages, we learned how classes can inherit behavior from parents. But sometimes, you don't want a "parent-child" relationship; you just want to ensure that completely different types of objects all follow the same set of rules. In Java, this is achieved through an Interface. An interface is a 100% abstract "contract" that specifies what a class must do, but not how it does it. It allows for a level of decoupling and flexibility that is impossible with class inheritance alone.

1. Defining and Implementing Interfaces

An interface is defined using the interface keyword. By default, all methods in an interface are public and abstract. A class "signs" the contract by using the implements keyword.


public interface Drawable {
    void draw(); // Abstract method (no body)
}

public class Circle implements Drawable {
    @Override
    public void draw() {
        System.out.println("Drawing a circle...");
    }
}

public class Button implements Drawable {
    @Override
    public void draw() {
        System.out.println("Drawing a UI button...");
    }
}

2. Multiple Implementation

One of the biggest advantages of interfaces over classes is that a Java class can implement multiple interfaces. This allows an object to "wear many hats" and be compatible with many different systems.


public class SmartPhone implements Camera, Phone, Computer {
    // Must implement methods from ALL three interfaces
}

3. Modern Interfaces: Default and Static Methods

Since Java 8, interfaces have become even more powerful. They can now include default methods (which have a body and are optional to override) and static methods (which belong to the interface itself).


public interface Drawable {
    void draw();

    default void displayInfo() {
        System.out.println("This is a drawable object.");
    }
}

Real-World Relationship: The Standardized Plug and the Legal Contract

Think of an Interface like the USB Standard. The "USB Interface" defines exactly what the physical plug looks like and how data is transferred. It doesn't care if the device is a mouse, a keyboard, or a hard drive. As long as the device "implements" the USB interface, it will work with your computer. This is the essence of Decoupling. Think of the implements keyword like signing a Legal Contract. When you sign a contract to be a "Driver," the contract doesn't tell you how to move your arms or legs (the Implementation); it just says you must be able to steer() and brake(). Whether you drive a truck or a sports car, you are compatible with the role because you fulfilled the contract's requirements.
