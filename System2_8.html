System Design II: Page 8 - Integration: Webhooks

Event-Driven Notifications

A Webhook is a simple but powerful mechanism: it’s a way for one application to provide other applications with real-time information. Instead of your server constantly asking another server "is there any new data?" (a wasteful process called polling), the other server simply "pushes" data to a URL you provide the moment an event occurs. This makes your application reactive and significantly reduces unnecessary network traffic.

1. Webhook Lifecycle

  • Subscription: You give the provider (e.g., Stripe, GitHub) a public URL on your server.
  • Trigger: An event occurs on their side (e.g., payment received, code pushed).
  • Push: The provider sends an HTTP POST request with a JSON payload to your URL.
  • Response: Your server processes the data and sends back an HTTP 200 OK to acknowledge receipt.

# Provider POST request to your endpoint
POST /api/webhooks/stripe
{ "event": "payment_succeeded", "amount": 1000 }
