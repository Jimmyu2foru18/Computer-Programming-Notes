C# I: Page 3 - Operators & Boolean Logic

The Engine of Calculation

In the previous page, we learned how to store data in variables. But a program that only stores data is just a static list. To create an Application, you must be able to manipulate that data—add numbers, compare values, and combine logical conditions. In C#, this is done through Operators. These special symbols tell the compiler to perform specific mathematical or logical tasks. C# operators are highly optimized and follow strict rules of "Precedence," ensuring that your calculations are predictable and accurate.

1. Arithmetic & Assignment Operators

C# supports all standard mathematical operations, along with "Compound Assignment" operators that make your code more concise.

  • +, -, *, /, %: Addition, Subtraction, Multiplication, Division, and Modulo (remainder).
  • +=, -=, *=, /=: Performs the operation and assigns the result back to the variable.
  • ++, --: Increments or decrements a value by exactly 1.

int balance = 100;
balance += 50; // balance is now 150
balance++;     // balance is now 151
Expert Note: In C#, dividing two integers results in an integer (e.g., 10 / 3 = 3). If you need a decimal result, at least one number must be a double or decimal (e.g., 10.0 / 3 = 3.333).

2. Comparison & Logical Operators

Comparison operators compare two values and return a bool. Logical operators are used to combine multiple boolean checks.

  • == (Equal to), != (Not equal to)
  • >, <, >=, <=
  • && (Logical AND): Both sides must be true.
  • || (Logical OR): At least one side must be true.
  • ! (Logical NOT): Reverses the boolean value.

int age = 20;
bool hasLicense = true;
bool canDrive = (age >= 18) && hasLicense; // true

3. Short-Circuit Evaluation

C# uses Short-Circuiting for its logical operators. If the result of an expression can be determined by evaluating the first part, the second part is never even looked at. This is a massive performance boost and can prevent errors (e.g., checking if an object is not NULL before accessing its property in the same line).

Real-World Relationship: The Cashier and the Bouncer

Think of Arithmetic Operators like a Cashier at a Grocery Store. They take items (Variables), add them up (+), apply a discount (-), and give you a total (Assignment). Think of Logical Operators like a Bouncer at a Club. The bouncer has a two-step check: (Has ID AND Is Over 21). The Short-Circuit is the bouncer saying, "I don't even need to see your ID if you're wearing a high school uniform." He stops the check early (efficiency). This logical "filtering" is what allows software to handle complex real-world rules with speed and accuracy.
