Python I: Page 7 - Collection Types: Dictionaries (dict)

Labeled Data: The Python Dictionary

In our previous pages, we looked at collections that are ordered by their position (Lists and Tuples) or defined by their uniqueness (Sets). However, in many real-world scenarios, we need to store data with labels. For example, if you are storing information about a student, a list like ["Alice", 20, "Computer Science"] is hard to read—you have to remember that index 1 is the age. A Dictionary solves this by storing data in Key-Value Pairs.

Dictionaries are unordered (in older Python versions) or order-preserving (in Python 3.7+), and they are optimized for extremely fast data retrieval. You look up a value using its unique "key," just like you'd look up a word in a physical dictionary.


# Creating a dictionary
student = {
    "name": "Alice",
    "age": 20,
    "major": "Computer Science"
}

# Accessing values
print(student["name"])  # Output: Alice

Safe Lookups and Manipulation

One common problem with dictionaries is trying to access a key that doesn't exist, which causes a KeyError. Python provides the .get() method for "safe" access.


# Safe access with a default value
email = student.get("email", "No email provided")
print(email)  # Output: No email provided

# Adding or updating data
student["email"] = "alice@university.edu"
student["age"] = 21

Dictionary Comprehensions

Just like lists, you can create dictionaries using a concise, one-line syntax. This is perfect for transforming data, such as creating a lookup table of squares or converting temperatures.


# Syntax: {key_expression: value_expression for item in iterable}
squares_dict = {x: x**2 for x in range(5)}
# Output: {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}

Iterating Through Dictionaries

You can loop through a dictionary in several ways, depending on whether you need the keys, the values, or both.

  • for key in student: Loops through only the keys.
  • for value in student.values(): Loops through only the values.
  • for key, value in student.items(): The most common way—loops through both simultaneously.

Real-World Relationship: The Library Catalog and the Smartphone Contact List

Think of a Dictionary like a Library Catalog. You don't walk through the library counting books until you find "Moby Dick" (that would be a List search). Instead, you look up the unique ISBN number (the Key), and it tells you exactly which shelf the book (the Value) is on. Another example is your Smartphone Contact List. You don't remember people by their index number ("Friend #42"); you remember them by their Name (the Key). When you tap a name, the phone "returns" their phone number, email, and photo (the Values). Dictionaries allow you to organize your data logically so that you can find what you need instantly, no matter how large the collection becomes.
